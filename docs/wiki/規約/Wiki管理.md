# 規約: Wiki 管理

`docs/wiki/` 配下のドキュメント管理ルール。

GitHub Wiki ではなく **本体リポジトリの `docs/wiki/`** で管理し、GitHub Pages 配信 + raw URL 取得 + PR ベースのレビューを実現する。

## フォルダ構成原則

### トップ構造

```
docs/
├── index.html              ← サイトトップハブ（既存）
├── mock/                   ← モック画面（HTML、既存）
└── wiki/                   ← ドキュメント（本ルール対象）
    ├── README.md           ← Wiki トップ + 全ドキュメントツリー
    ├── テンプレート/       ← 実体ファイルの書式定義
    ├── 規約/               ← プロセス・手順・ルール
    ├── チェックシート/     ← やるべき作業の完了確認シート
    ├── 判定フローチャート/ ← 判定フローチャート・判定表
    ├── 状態遷移/           ← Issue/PR の状態遷移図
    ├── 観点/               ← 要件洗い出し観点集
    ├── 設計図/             ← プロジェクト固有の設計図（任意）
    ├── 外部API/            ← 外部 API リファレンス（任意）
    └── 外部ライブラリ/     ← 外部ライブラリリファレンス（任意）
```

## README.md / index.html の使い分け

| ファイル | 主用途 | 表示タイミング |
| --- | --- | --- |
| `README.md` | 目次のみ | GitHub UI でディレクトリを開いた時に **自動表示** |
| `index.html` | リッチな見た目（デザイン付きハブ） | GitHub Pages 配信時のディレクトリデフォルト |

### README.md の中身

- **README.md には目次だけを書く**。使い方説明・共通ルール・追加手順・補足リンク等の本文は README.md には置かない
- 目次は「タイトル + `## 目次` 見出し + 表」だけの構成
- 表の列は最低限「No / ページ / 概要」、必要に応じて「対象作業」等を追加

### index.html を使うケース

- サイトトップ（`docs/index.html`）のようにデザインが要る
- 動的要素（JS / フォーム）が要る
- リッチな配色 / レイアウトが必要

通常のドキュメントページはすべて Markdown で書く（HTML タグ原則不使用）。

## ファイル命名規約

| 規約 | 内容 | 例 |
| --- | --- | --- |
| ファイル名は **短く** | フォルダで分類されるためプレフィックス不要 | `テンプレート.md` |
| 言語 | 日本語ファイル名 OK | `モジュール構成.md` |
| 拡張子 | `.md` 統一 | - |
| インデックス | `README.md` 固定 | - |
| 空白 | アンダースコア or キャメルで回避 | `外部API一覧.md` |

## 内部リンクの書き方

| 種別 | 書き方 | 補足 |
| --- | --- | --- |
| 同ディレクトリ | `[表示](./xxx.md)` | `.md` 拡張子付きで OK |
| 親ディレクトリ | `[表示](../README.md)` | 相対パス |
| 別ディレクトリ | `[表示](../規約/コメント.md)` | 相対パス |
| ディレクトリ自体 | `[表示](./サブフォルダ/)` | README.md が自動表示される |
| 外部 URL | `[表示](https://...)` | - |
| 画像 | `![alt](./images/xxx.png)` | 各階層に `images/` 配下に置く |


## 新規ページ追加時の手順

| No | 手順 | 補足 |
| --- | --- | --- |
| 1 | 適切なフォルダを決定（既存カテゴリ or 新規フォルダ） | 業務ドメイン軸で判断 |
| 2 | ファイルを作成（`.md` 拡張子、短い名前） | プレフィックス不要 |
| 3 | 同階層 README.md の一覧表に 1 行追加 | - |
| 4 | ルート `docs/wiki/README.md` の全ドキュメントツリーにも 1 行追加 | 手動更新 |
| 5 | 新規フォルダ作成時は **README.md を必ず併置** | ナビゲーション用 |

## 更新時の手順

| No | 手順 | 補足 |
| --- | --- | --- |
| 1 | 該当ファイルを直接編集 | - |
| 2 | 内部リンクが他ファイルから貼られている場合は影響範囲確認 | `grep -r "(./xxx.md)" docs/wiki/` |
| 3 | リネーム時は全 README + 全 grep ヒットを置換 | - |
| 4 | 削除時は履歴を git log に任せる（履歴コメントを残さない） | - |
