# ai-monitor: デザインスタイル規約

プロジェクト初期に「画面全体の雰囲気・方向性」を決めるための材料集。
スタイル名と短い概要だけを並べる。
配色・フォント・角丸の数値などの実装詳細はここでは扱わず、プロジェクト側 Wiki（`ai-monitor テンプレート: 画面デザイン` を継承したページ）で決める。

このページは網羅を狙うが、プロジェクトの要件次第で追加・改変してよい。

## スタイル一覧

| スタイル名 | 概要 | 向く用途 |
| --- | --- | --- |
| フラット | 装飾を排し、面と色で情報を区切る。影や立体感を持たない | 業務系・情報密度が高い画面 |
| マテリアル | 紙のメタファで影と階層を表現する。明確な立体感を持つ | 一般消費者向け・操作の手応えを重視 |
| モダン | フラットとマテリアルの折衷。控えめな影と適度な余白 | SaaS の標準・迷ったらここ |
| ミニマル | 要素を極限まで減らす。余白を主役にする | ブランド訴求・ポートフォリオ |
| ニューモーフィズム | 同色のソフトな凹凸で要素を表現する | 単機能アプリ・実験的 UI |
| ガラスモーフィズム | 半透明とぼかしで奥行きを出す | ダッシュボード・モバイル系 |
| スキューモーフィズム | 現実の物体（紙・革・木目）を模倣する | 学習系・ライフスタイル系 |
| レトロ | 古い時代の意匠を引用する（90 年代 Web / 80 年代テクノなど） | エンタメ・ニッチ層向け |
| ブルータリズム | 飾らず、生のままの要素を強調する。あえて素朴・粗野に | 個性訴求・アート系 |
| ターミナル風 | 等幅フォント・モノクロまたは限定色で CLI を模倣する | 開発者向けツール |
| ダーク | 暗い背景を主にし、目に優しい配色を組む | 長時間利用・夜間利用が多いツール |
| エディトリアル | 雑誌・新聞の組版に倣う。タイポグラフィを主役にする | コンテンツ閲覧・読み物系 |
| カード中心 | カードを基本ユニットにし、画面全体をカードの集まりで構成する | フィード・ダッシュボード |
| コンパクト業務系 | 余白を抑え、情報密度を最優先する | 管理画面・社内ツール |
| カラフル / ポップ | 鮮やかな配色と曲線を多用する | コンシューマ・若年層向け |

## 組み合わせの考え方

- 基本は 1 つを採用する
- 「ダーク × ターミナル風」「モダン × カード中心」のように、性質が重ならない 2 つの組み合わせは可
- 「フラット × マテリアル」のように矛盾する組み合わせは避ける
