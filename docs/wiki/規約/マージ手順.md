# ai-monitor: マージ手順

各レイヤーの conductor が自分の配下 PR を base ブランチへマージするときの共通手順。
subsystem → story / story → epic / epic → master の全昇格マージで同じ。
conductor スキルに動的コンテキストとして注入して使う。

## 手順

1. 対象 PR の worktree に復帰し、base ブランチを取り込む（`git fetch` + `git merge origin/{base}`）
2. コンフリクトがないことを確認する（あれば「コンフリクト時の対応」へ）
3. `gh pr merge --squash --delete-branch` で squash マージ + リモートブランチ削除
4. ローカルの worktree を削除する
5. PR が merged 状態・リモートブランチが削除済みであることを確認する（紐づく Issue はマージで自動 close）

## コンフリクト時の対応

1. 対象 PR に競合ファイルと「どちらの変更を残すか」の相談コメントを投稿 + `議論中` 付与 + `assignee=ユーザー` 設定
2. 応答ループ: ユーザーの判断コメント + assignee 外し → 指示どおり解消 commit → 解消内容を同スレッドに返信追記（追加の競合・疑問があれば再相談 + `assignee=ユーザー` 再設定）
3. 全競合解消後、手順 3（squash マージ）に合流する
