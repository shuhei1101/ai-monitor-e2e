# 分解判定（応答ループ）

ユーザーのフィードバックを受けてサブ Issue 案を調整し、再び待機に入る。

## 手順

### 未処理コメントの取得

MCP `list_addressed_comments` を呼ぶ:
- `number`: $issue_number
- `is_pr`: false
- `addressee`: `intake-issue-triager`
- `include_resolved`: false

### 内容に応じた返信・案の修正

案の更新は「案を {変更後} に修正しました」と短く追記する。
案の内容変更は Issue 本文には反映しない（案はコメント上の議論として管理し、承認後に「サブIssue起票（完了処理）」で起票する）。

追記は MCP `reply_comment` を呼ぶ:
- `comment_node_id`: 返信対象コメントの node_id
- `sender`: `intake-issue-triager`
- `receiver`: ユーザーログイン名
- `body`: 返信本文

### assignee 再設定

MCP `set_assignee` を呼ぶ:
- `number`: $issue_number
- `is_pr`: false
