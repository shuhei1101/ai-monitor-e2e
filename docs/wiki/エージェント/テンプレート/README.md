# フェーズ索引

## セクション一覧

| セクション | 必須or条件 |
| --- | --- |
| 冒頭リード | 必須 |

## `冒頭リード`

### 記述例

```markdown
# story-conductor
```

### 補足

- エージェント名（スキルフォルダ名・tmux セッション名と一致）の 1 行のみ。
  役割の説明は SKILL.md の冒頭リードが持つ

## `## フェーズ一覧`

### 記述例

```markdown
## フェーズ一覧

| フェーズ | 起動条件 |
| --- | --- |
| [初期処理](./フェーズ/初期処理.md) | 毎ターン最初に必ず実行（以降の判定材料を取得） |
| [要件確定（初回）](./フェーズ/要件確定（初回）.md) | 完了報告コメント未投稿（未作業） |
| [要件確定（応答ループ）](./フェーズ/要件確定（応答ループ）.md) | `議論中` あり + 未解決の自分宛コメントがある |
| [要件確定（完了処理）](./フェーズ/要件確定（完了処理）.md) | `議論中` なし + 完了報告コメント投稿済み + story Draft PR 未作成 |
| [子subsystem起票](./フェーズ/子subsystem起票.md) | 未解決の自分宛コメントに single-scenario-writer のシナリオ設計完了報告がある |
```

### 補足

**フェーズ列:**
- フェーズ名は内容の日本語名で書く（`{UC 名}` / `{UC 名}（初回）` / `{UC 名}（応答ループ）` / `{UC 名}（完了処理）` 等）
- `[{フェーズ名}](./フェーズ/{フェーズ名}.md)` のリンクで書き、ファイル名と一致させる

**起動条件列:**
- 未解決の自分宛コメント（誰から・報告種別）・`議論中` の有無・本文や成果物の状態で書く
- 上から順に判定して最初にマッチした 1 フェーズだけが実行されるため、行間で重複しない排他条件で書く。
  未作業の初回フェーズは「{成果物}未投稿（未作業）」のように成果物の不在を条件にする

**補足:**
- 先頭に [初期処理](./フェーズ/初期処理.md) の行を必ず置く（起動条件は「毎ターン最初に必ず実行」で固定）
- 2 行目以降はフェーズの時系列順（初回 → 応答ループ → 完了処理 → 復帰系）に並べる
