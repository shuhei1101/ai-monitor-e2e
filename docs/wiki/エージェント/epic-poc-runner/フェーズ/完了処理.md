# 完了処理

承認された検証結果を親 epic Issue に記録し、確認ラベルを epic-conductor に渡して役割を終える。
PoC PR・ブランチは保持したまま（closed PR の diff が恒久記録になる。close は epic-conductor の PoC 結果確認フェーズが行う）。

## 手順

### 自分宛コメントの選別

未回答・未対応の自分宛コメントが残る場合は「応答ループ」に戻ってユーザーに確認質問を投げる（以降の手順は実行しない）。

### 一括 Resolve

MCP `resolve_comments` を呼ぶ:
- `node_ids`: 自分宛コメントの `node_id` 配列

### 親 epic Issue への結果記録

$pr_number の PoC PR 本文の `## 紐づく Issue` から親 epic Issue 番号を得る。
親 epic Issue の本文を取得（MCP `get_issue_or_pr`）し、テンプレート（イシュー本文/エピック）に沿って `## PoC 結果` セクションを追記または更新する。

- 検証構成 / 成功条件 / 結果（成立 or 不成立）/ PoC PR リンクを記載する

MCP `update_body` を呼ぶ:
- `number`: 親 epic Issue 番号
- `is_pr`: false
- `body`: `## PoC 結果` を追記・更新した本文

### 完了報告の投稿

MCP `comment` を呼ぶ:
- `number`: 親 epic Issue 番号
- `is_pr`: false
- `sender`: `epic-poc-runner`
- `receiver`: `epic-conductor`
- `body`: PoC 検証の完了報告（成立 or 不成立の結論 + PoC PR リンク + 親 epic 本文への記録内容の要約）

### 確認ラベルの引き継ぎ

MCP `add_labels` を呼ぶ:
- `number`: 親 epic Issue 番号
- `is_pr`: false
- `labels`:
  - `$AI_MONITOR_LABEL_CONFIRM_EPIC_CONDUCTOR` の値

### ラベル除去

MCP `transition_phase` を呼ぶ:
- `number`: $pr_number
- `is_pr`: true
- `remove_labels_`:
  - `$AI_MONITOR_LABEL_CONFIRM_EPIC_POC_RUNNER` の値
- `add_labels_`: なし（PoC PR・ブランチは保持したまま）

### 作業完了報告

MCP `report_completion` を呼ぶ:
- `agent_name`: `epic-poc-runner`
- `number`: $pr_number
