# 検証実行

方針固めで確定した検証構成に沿って PoC コードを実装・実行し、結果を本文に記録して待機に入る。
案が複数ある場合は案ごとに追加 PoC PR を作成し、Task ツールでサブエージェントを並列起動して実装・実行する。

## 手順

### 自分宛コメントの選別

未回答・未対応の自分宛コメントが残る場合は「応答ループ」に戻ってユーザーに確認質問を投げる（以降の手順は実行しない）。

### 一括 Resolve

MCP `resolve_comments` を呼ぶ:
- `node_ids`: 自分宛コメントの `node_id` 配列

### 追加 PoC PR の作成（案が複数ある場合のみ）

方針固めで確定した検証構成が複数案ある場合、案ごとに追加 PoC PR を作成する（1 案 = 1 PR）。案が 1 つなら本手順はスキップ。

各追加案について実施する:

1. MCP `worktree_create` を呼ぶ（`branch`: `poc/epic/{ドメイン}/{テーマ}-{案名}`）
2. MCP `create_draft_pr` を呼ぶ（`head_branch`: 作成したブランチ・`base_branch`: `master`・`title`: `PoC: {検証テーマ}（epic #{親epic番号}・{案名}）`・`body`: `## 紐づく Issue` + 案固有の `## リスク仮説` / `## 検証構成` / `## 成功条件`）
3. MCP `add_watch_targets` を呼ぶ（`agent_name`: `epic-poc-runner`・`number`: $pr_number・`watch_numbers`: 作成した追加 PoC PR の番号）

### 検証コード実装・実行

案の数に応じて分岐する。

**案が 1 つの場合:**

$pr_number の PoC PR のブランチで検証コードを実装・push し、実行する。

**案が複数の場合（Task ツールでサブエージェント並列起動）:**

各 PoC PR（$pr_number + 追加 PoC PR 群）について、Task ツールで 1 サブエージェント = 1 案の担当を並列起動する。
各サブエージェントは以下を担う:

- 担当 PoC PR のブランチで検証コードを実装・push
- 実行して実測値を収集
- 実測値と最小再現コードをメインへ返す

メインセッションは全サブエージェントの完了を待ち、結果を集約する。

### 検証結果の記録

各 PoC PR 本文に `## 検証結果`（実測値表 + 所感）と `## 最小再現コード`（核心部 10〜30 行 + diff の見どころ）を記入する。

各 PR について MCP `update_body` を呼ぶ:
- `number`: 対象 PoC PR の番号
- `is_pr`: true
- `body`: 既存本文 + 追記した 2 セクション

### 結果報告の投稿

$pr_number の PoC PR に結果報告コメントを投稿する（追加 PoC PR がある場合は各 PR の実測値サマリを含める）。

MCP `comment` を呼ぶ:
- `number`: $pr_number
- `is_pr`: true
- `sender`: `epic-poc-runner`
- `receiver`: ユーザーログイン名
- `body`:
  - 成立の場合: 実測値サマリ表 + 判定 + 所感 + 追加検証の要否確認
  - 不成立の場合: 実測値サマリ表 + 全案 ❌ の結論 + 代替案 or 中止の相談

### 議論中 付与 + 待機

MCP `add_labels` を呼ぶ:
- `number`: $pr_number
- `is_pr`: true
- `labels`:
  - `$AI_MONITOR_LABEL_IN_DISCUSSION` の値

続けて MCP `set_assignee` を呼ぶ:
- `number`: $pr_number
- `is_pr`: true

### 作業完了報告

MCP `report_completion` を呼ぶ:
- `agent_name`: `epic-poc-runner`
- `number`: $pr_number
