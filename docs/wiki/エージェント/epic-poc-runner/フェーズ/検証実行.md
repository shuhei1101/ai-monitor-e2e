# 検証実行

方針固めで確定した検証構成に沿って PoC コードを実装・実行し、結果を本文に記録して待機に入る。
1 PoC PR = 1 検証構成の単一案運用（代替案が必要になった場合は応答ループで `## 検証構成` を書き換えて再検証する）。

## 手順

### 自分宛コメントの選別

未回答・未対応の自分宛コメントが残る場合は「応答ループ」に戻ってユーザーに確認質問を投げる（以降の手順は実行しない）。

### 一括 Resolve

MCP `resolve_comments` を呼ぶ:
- `node_ids`: 自分宛コメントの `node_id` 配列

### 検証コード実装・実行

$pr_number の PoC PR のブランチで検証コードを実装・push し、実行して実測値を収集する。

### 検証結果の記録

PoC PR 本文に `## 検証結果`（実測値表 + 所感）と `## 最小再現コード`（核心部 10〜30 行 + diff の見どころ）を記入する。

MCP `update_body` を呼ぶ:
- `number`: $pr_number
- `is_pr`: true
- `body`: 既存本文 + 追記した 2 セクション

### 結果報告の投稿

MCP `comment` を呼ぶ:
- `number`: $pr_number
- `is_pr`: true
- `sender`: `epic-poc-runner`
- `receiver`: ユーザーログイン名
- `body`:
  - 成立の場合: 実測値サマリ表 + 判定 + 所感 + 追加検証の要否確認
  - 不成立の場合: 実測値サマリ表 + 不成立の結論 + 代替案 or 中止の相談

### 議論中 付与 + 待機

MCP `add_labels` を呼ぶ:
- `number`: $pr_number
- `is_pr`: true
- `labels`:
  - `$AI_MONITOR_LABEL_IN_DISCUSSION` の値

続けて MCP `set_assignee` を呼ぶ:
- `number`: $pr_number
- `is_pr`: true

### 作業完了報告

MCP `report_completion` を呼ぶ:
- `agent_name`: `epic-poc-runner`
- `number`: $pr_number
