# 応答ループ

待機中のユーザーフィードバックに対応し、再び待機に入る。
どの待機（要件確定・PoC 疑問・バグ差し戻しの方針・マージのコンフリクト相談）かは、未解決の自分宛スレッドの文脈で判断する。

## 手順

### 内容に応じた対応

| 待機の文脈 | 対応 |
| --- | --- |
| 要件確定の本文修正指示 | 本文を修正（`update_body`）して「本文の `## XXX` を更新しました」と短く追記 |
| PoC 疑問への回答 | 回答を踏まえて「PoC結果確認」の続行 / 再検証の判断材料として追記 |
| バグ差し戻しの方針調整 | 方針案コメントを更新して短く追記 |
| コンフリクト解消の判断 | 指示どおり解消 commit + 解消内容を同スレッドに返信追記 |

追記は MCP `reply_comment` を呼ぶ:
- `comment_node_id`: 返信対象コメントの node_id
- `sender`: `epic-conductor`
- `receiver`: ユーザーログイン名
- `body`: 返信本文

### assignee 再設定

MCP `set_assignee` を呼ぶ:
- `number`: $issue_number
- `is_pr`: false

### 作業完了報告

MCP `report_completion` を呼ぶ:
- `agent_name`: `epic-conductor`
- `number`: $issue_number
