# 要件確定（完了処理）

承認された要件をもとに、PoC 要否・画面変更有無の回答に応じて Draft PR を作成し、次フェーズの担当へ引き継ぐ。

## 手順

### 自分宛コメントの選別

未回答・未対応の自分宛コメントが残る場合は「応答ループ」に戻ってユーザーに確認質問を投げる（以降の手順は実行しない）。

### 一括 Resolve

MCP `resolve_comments` を呼ぶ:
- `node_ids`: 自分宛コメントの `node_id` 配列

### Draft PR の作成

要件確定で確定した回答で分岐する。

**PoC 必要の場合**（PoC Draft PR を作成。epic Draft PR は作成しない）:

MCP `worktree_create` を呼ぶ:
- `branch`: `poc/epic/{ドメイン}/{検証テーマ}`

続けて MCP `create_draft_pr` を呼ぶ:
- `head_branch`: 作成したブランチ
- `base_branch`: `master`
- `title`: `PoC: {検証テーマ}（epic #$issue_number）`
- `body`: `## 紐づく Issue` のみ（`- #$issue_number`）

**PoC 不要の場合**（epic Draft PR を作成）:

MCP `worktree_create` を呼ぶ:
- `branch`: `{type}/epic/{ドメイン}`（type は intake で判定済みの `type:*` に対応）

続けて MCP `create_draft_pr` を呼ぶ:
- `head_branch`: 作成したブランチ
- `base_branch`: `master`
- `title`: epic のタイトル
- `body`: `## 紐づく Issue` のみ（`- #$issue_number`）

### 監視面の登録

MCP `add_watch_targets` を呼ぶ:
- `agent_name`: `epic-conductor`
- `number`: $issue_number
- `watch_numbers`: 作成した PR の番号

### 次フェーズの割り当て

作成した PR に次の確認ラベルを付与する（MCP `add_labels`・`is_pr`: true）。

| 分岐 | 付与するラベル | 追加の操作 |
| --- | --- | --- |
| PoC 必要 | `$AI_MONITOR_LABEL_CONFIRM_EPIC_POC_RUNNER` の値 | - |
| PoC 不要・画面変更あり | `$AI_MONITOR_LABEL_CONFIRM_MOCK_DESIGNER` の値 | MCP `comment`（`is_pr`: true・`receiver`: `mock-designer`）で画面方針の要点を指示コメント投稿 |
| PoC 不要・画面変更なし | `$AI_MONITOR_LABEL_CONFIRM_COMPLEX_SCENARIO_WRITER` の値 | - |

### ラベル除去

MCP `transition_phase` を呼ぶ:
- `number`: $issue_number
- `is_pr`: false
- `remove_labels_`:
  - `$AI_MONITOR_LABEL_CONFIRM_EPIC_CONDUCTOR` の値
- `add_labels_`: なし（次の呼び出しは配下からの完了報告で復帰する）

### 作業完了報告

MCP `report_completion` を呼ぶ:
- `agent_name`: `epic-conductor`
- `number`: $issue_number
