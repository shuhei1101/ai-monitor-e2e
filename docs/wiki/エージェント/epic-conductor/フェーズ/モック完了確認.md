# モック完了確認

mock-designer の完了報告を受けて epic PR の UI 設計を確認し、複合シナリオ設計へ引き継ぐ。
モック・UI 設計は全体UI設計のユーザー承認ゲートで合意済みのため、ユーザー承認なしの自動完了で次へ進める。

## 手順

### UI 設計の確認

epic PR 本文の `## UI 設計`（画面一覧 / 画面遷移 / モック URL）と mock-designer の完了報告コメントを確認する。

### 完了報告の Resolve と引き継ぎ

MCP `resolve_comments` で完了報告コメントを Resolve する。

続けて MCP `add_labels` を呼ぶ:
- `number`: epic PR の番号
- `is_pr`: true
- `labels`:
  - `$AI_MONITOR_LABEL_CONFIRM_COMPLEX_SCENARIO_WRITER` の値

### ラベル除去

MCP `transition_phase` を呼ぶ:
- `number`: $issue_number
- `is_pr`: false
- `remove_labels_`:
  - `$AI_MONITOR_LABEL_CONFIRM_EPIC_CONDUCTOR` の値
- `add_labels_`: なし（子 story 起票時に complex-scenario-writer が再付与する）

### 作業完了報告

MCP `report_completion` を呼ぶ:
- `agent_name`: `epic-conductor`
- `number`: $issue_number
