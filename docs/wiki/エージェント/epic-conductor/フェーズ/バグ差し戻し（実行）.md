# バグ差し戻し（実行）

承認された対応方針で該当 story Issue へバグを差し戻す。
指揮系統は 1 段ずつ辿る（該当 subsystem への中継は story-conductor が行う）。
進行中のバグは master 未反映のため、新規 Issue は起票せず該当 story Issue を reopen する。

## 手順

### story への差し戻し

方針案の「対応要」の各 story について実施する。

1. MCP `reopen_issue` を呼ぶ（`number`: 該当 story Issue 番号）
2. MCP `comment` を呼ぶ（`number`: 該当 story Issue 番号・`is_pr`: false・`sender`: `epic-conductor`・`receiver`: `story-conductor`・`body`: fail 内容 + 修正方針）
3. MCP `add_labels` を呼ぶ（`number`: 該当 story Issue 番号・`is_pr`: false・`labels`: `$AI_MONITOR_LABEL_CONFIRM_STORY_CONDUCTOR` の値）

### 失敗報告への返信と Resolve

MCP `reply_comment` を呼ぶ:
- `comment_node_id`: complex-scenario-writer の失敗報告コメントの node_id
- `sender`: `epic-conductor`
- `receiver`: `complex-scenario-writer`
- `body`: 差し戻し結果（対象 story Issue 番号）

続けて MCP `resolve_comments` で自分宛コメントを一括 Resolve する。

### ラベル除去

MCP `transition_phase` を呼ぶ:
- `number`: $issue_number
- `is_pr`: false
- `remove_labels_`:
  - `$AI_MONITOR_LABEL_CONFIRM_EPIC_CONDUCTOR` の値
- `add_labels_`: なし（バグ修正後の再テストは統合テスト起動の経路で再委任される）

### 作業完了報告

MCP `report_completion` を呼ぶ:
- `agent_name`: `epic-conductor`
- `number`: $issue_number
