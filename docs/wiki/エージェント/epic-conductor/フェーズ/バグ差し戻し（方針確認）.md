# バグ差し戻し（方針確認）

complex-scenario-writer の失敗報告（実装側の問題）を受けて、対応方針案をユーザーに確認する。
実装修正は設計書の修正を伴うため、差し戻し前に必ずユーザーと方針を合意する。

## 手順

### 対応方針案の投稿

失敗内容から対応方針案を組み立てて投稿する。

MCP `comment` を呼ぶ:
- `number`: $issue_number
- `is_pr`: false
- `sender`: `epic-conductor`
- `receiver`: ユーザーログイン名
- `body`:
  ```
  複合 UC テストの失敗を受けた対応方針案です。

  | 修正対象 story | 修正内容の要約 | 対応要否 |
  | --- | --- | --- |
  | #{番号} {story 名} | {要約} | {要 / 不要} |

  - 問題なければ `議論中` ラベルを外す → 該当 story へ差し戻します
  - 調整が必要なら assignee を外してフィードバックコメントを記入
  ```

### 議論中 付与 + 待機

MCP `add_labels` を呼ぶ:
- `number`: $issue_number
- `is_pr`: false
- `labels`:
  - `$AI_MONITOR_LABEL_IN_DISCUSSION` の値

続けて MCP `set_assignee` を呼ぶ:
- `number`: $issue_number
- `is_pr`: false

### 作業完了報告

MCP `report_completion` を呼ぶ:
- `agent_name`: `epic-conductor`
- `number`: $issue_number
