# epic-conductor

## フェーズ一覧

判定は毎ターン、最新の取得結果（本文・ラベル・自分宛の未解決コメント）を材料に、上から順に見て最初にマッチしたフェーズを 1 つ実行する。

| フェーズ | 起動条件 |
| --- | --- |
| [初期処理](./フェーズ/初期処理.md) | 毎ターン最初に必ず実行（以降の判定材料を取得） |
| [要件確定（初回）](./フェーズ/要件確定（初回）.md) | 本文の 5 セクション未記入（未作業） |
| [応答ループ](./フェーズ/応答ループ.md) | `議論中` あり + 未解決の自分宛コメントがある |
| [要件確定（完了処理）](./フェーズ/要件確定（完了処理）.md) | `議論中` なし + 本文確定済み + epic / PoC Draft PR 未作成 |
| [PoC結果確認](./フェーズ/PoC結果確認.md) | 未解決の自分宛コメントに epic-poc-runner の完了報告、または PoC 疑問スレッドへのユーザー回答がある |
| [モック完了確認](./フェーズ/モック完了確認.md) | 未解決の自分宛コメントに mock-designer の完了報告がある |
| [子story起票](./フェーズ/子story起票.md) | 未解決の自分宛コメントに complex-scenario-writer のシナリオ設計完了報告がある |
| [統合テスト起動](./フェーズ/統合テスト起動.md) | 未解決の自分宛コメントに story-conductor の完了報告（story 完了）がある |
| [バグ差し戻し（方針確認）](./フェーズ/バグ差し戻し（方針確認）.md) | 未解決の自分宛コメントに complex-scenario-writer の失敗報告があり、対応方針案コメント未投稿 |
| [バグ差し戻し（実行）](./フェーズ/バグ差し戻し（実行）.md) | `議論中` なし + 対応方針案コメント投稿済み（未 Resolve） |
| [epicマージ](./フェーズ/epicマージ.md) | 未解決の自分宛コメントに complex-scenario-writer の完了報告（全 pass）がある |
