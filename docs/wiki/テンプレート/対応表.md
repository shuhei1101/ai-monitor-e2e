# ai-monitor テンプレート: 対応表

エージェント × 参照ドキュメントの星取り表を管理する書式。
`read_agent_docs.py` が読み取って各スキルの起動時に ○ のドキュメントを動的注入する。

## ファイル構成

| ファイル | 役割 |
| --- | --- |
| `Claudeハーネス/共通対応表/{対象}対応表.md` | 全プロジェクト共通の星取り表（ai-monitor Wiki に置く） |
| `Claudeハーネス/対応表/{対象}対応表.md` | プロジェクト固有の星取り表（各プロジェクトの Wiki に置く） |

対応表は用途ごとにファイルを分ける（例: `エージェント参照ドキュメント対応表.md` = 共通ドキュメント / `エージェント言語規約対応表.md` = 言語規約 / `プロジェクトドキュメント対応表.md` = プロジェクト固有設計書）。
新設したファイルは置いた側の Wiki の `Claudeハーネス/README.md` の目次に追加する。

## 本文構成

H1 タイトルの直下に星取り表を 1 つ置く（セクション見出しは作らない）。

### 記述例

```markdown
# エージェント参照ドキュメント対応表

| ドキュメント | epic-conductor | story-conductor | architect | tester |
| --- | --- | --- | --- | --- |
| [規約/コメント.md](../../規約/コメント.md) | ○ | ○ | ○ | ○ |
| [規約/マージ手順.md](../../規約/マージ手順.md) | ○ | ○ | - | - |
| [python/core/命名規則.md](https://raw.githubusercontent.com/shuhei1101/my-plugins/master/docs/rules/python/core/命名規則.md) | - | - | ○ | ○ |
```

### 補足

**ドキュメント列:**
- リンクで書く。
  Wiki 内のページはラベル = Wiki 相対パス + 相対リンク、外部ドキュメントは絶対 URL のリンク
- 行の追加 = 注入候補の追加（`read_agent_docs.py` の変更は不要）

**エージェント列:**
- その表を消費するエージェントだけを列に並べる（全エージェントを揃える必要はない）

**マーク:**
- 注入する = `○` / 注入しない = `-`
