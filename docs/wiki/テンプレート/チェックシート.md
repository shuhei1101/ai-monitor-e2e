# ai-monitor テンプレート: チェックシート

`docs/wiki/チェックシート/{対象}.md` 形式のチェックシートを作るときの書式定義。

チェックシートは **やるべき作業が既に決まっていて、それが全部済んでいるか / 大丈夫か** を確認するためのシート。
「◯◯した」「◯◯は大丈夫」「◯◯が更新されている」のような **確認文** を並べる。
配置先は `docs/wiki/チェックシート/{対象}.md`。
新規作成時は同フォルダの `README.md` の目次に 1 行追加する。

## セクション一覧

| セクション | 必須or条件 | 担当 |
| --- | --- | --- |
| 冒頭リード | 必須 | 規約整備者 |
| `## チェック項目` | 必須 | 規約整備者 |

## `冒頭リード`

### 記述例

```markdown
# {対象作業} チェックシート

（このチェックシートが何を確認するツールか 1〜2 行）
```

### 補足

- 1 行目はページタイトル + `チェックシート` 接尾辞
- 何を完了確認するかを 3 秒で把握できる 1〜2 行で書く

## `## チェック項目`

### 記述例

```markdown
## チェック項目

| 項目 | 補足 |
| --- | --- |
| 単体テストが全部通っている | ローカル + CI 両方確認 |
| 関連ドキュメントを更新した | Wiki + Rules + CLAUDE.md |
| Draft を Ready に変換した | - |
| レビュー承認は 2 件以上ある | - |
```

### 補足

**「項目」列:**
- 「◯◯した」「◯◯は大丈夫」「◯◯が更新されている」のような **確認文** で書く
- YES / NO で答えられる 1 文
- 決定を含まない（決定するなら判定フローチャート / 判定表を使う）
- 完了ベース（動作を過去形で）で書くと迷いにくい

**「補足」列:**
- 追加の説明・実行環境・注意点を 1 行で
- 該当なしなら `-`

**並び順:**
- 実行順があるなら時系列で並べる
- 独立チェックなら重要度順で並べる
