# ai-monitor テンプレート: 判定フローチャート

`docs/wiki/判定フローチャート/{対象}.md` 形式の判定フローチャートを作るときの書式定義。

判定フローチャートは **決めたい内容が複数の判定軸で決まる** 場面で使う。

配置先は `docs/wiki/判定フローチャート/{対象}.md`。
新規作成時は同フォルダの `README.md` の目次に 1 行追加する。

## ファイル命名規則

判定したい選択肢 2 つを表す名前を `{対象}` に入れる。
以下のどちらかで命名する:

| パターン | 例 |
| --- | --- |
| 判定 2 択の **両方を包含する上位概念**（推奨） | `シナリオ.md`（複合ユースケース or 単一ユースケース の判定） |
| 判定 2 択の名前を **連結** して並記 | `複合ユースケース単一ユースケース.md` |

- 命名から「何を判定するフローチャートか」が一目で分かることが重要
- 判定対象が 3 択以上（例: `既存流用` / `既存修正` / `新規複合` / `新規単一`）の場合も、それらを包含する上位概念（`シナリオ`）を使う
- 「粒度」「レベル」等の抽象語だけは避ける（何の粒度 / 何のレベルか分からない）

## `タイトル`

### 記述例

```markdown
# {対象作業} 判定フローチャート
```

### 補足

- 1 行目はページタイトル（ファイル名と一致）+ `判定フローチャート` 接尾辞
- 冒頭リードは記載しない
