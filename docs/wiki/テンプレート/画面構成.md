# ai-monitor テンプレート: 画面構成

**1 画面 = 1 ファイル**で画面要素の仕様（表示条件・入力制限・初期値・取得元・アクション）を表で確定する。

## ファイル構成

`docs/wiki/設計図/画面構成/` 配下に **画面単位の .md をフラットに並べる**（サブフォルダは原則なし）。
README.md でカテゴリー別に分類して索引。

| ファイル | 役割 |
| --- | --- |
| `README.md` | カテゴリー別の索引 |
| `{画面名}.md` | 1 画面の要素仕様 |

ファイル名は **日本語の画面名**（例: `サインイン画面.md` / `タスク作成画面.md` / `タスク一覧画面.md`）。

## セクション一覧

| 対象ファイル | セクション | サブセクション | 必須or条件 | 担当 | 補足 |
| --- | --- | --- | --- | --- | --- |
| インデックス | 冒頭リード | - | 必須 | architect | 索引の説明 |
| インデックス | `## {カテゴリ名}` | - | 必須 | architect | カテゴリーごとの画面一覧 |
| 詳細 | 冒頭リード | - | 必須 | architect | 画面概要 + 画面パス / モック参照 |
| 詳細 | `## 要素一覧` | - | 必須 | architect | 要素仕様の 11 列表 |

## `冒頭リード`（インデックスファイル）

### 記述例

```markdown
# 画面構成

1 画面ごとの要素仕様集。
1 ファイル = 1 画面（表示条件・制限・初期値・取得元・アクションの確定値）。
```

### 補足

- 1 行目は `# 画面構成`
- カテゴリーは業務ドメイン軸（`認証` / `タスク` 等）

## `## {カテゴリ名}`（インデックスファイル）

### 記述例

```markdown
## タスク

| 画面 | リンク | 概要 | 補足 |
| --- | --- | --- | --- |
| タスク一覧画面 | [タスク一覧画面](./タスク一覧画面.md) | タスクの検索・一覧表示 | - |
| タスク作成画面 | [タスク作成画面](./タスク作成画面.md) | タスクの新規作成フォーム | - |
```

### 補足

**カテゴリ列（H2 見出し）:**
- 業務ドメイン軸（`認証` / `タスク` / `ユーザー` 等）

**リンク列:**
- `[表示](./{ファイル名}.md)` 形式

**概要列:**
- 1 行で画面の役割を要約

**補足:**
- 新規画面追加時は **必ずこの索引にも 1 行追加**（手動更新）

## `冒頭リード`（詳細ファイル）

### 記述例

```markdown
# タスク作成画面

画面: `app/(authenticated)/tasks/new/page.tsx`
モック: `docs/mock/pages/タスク作成/issues/12/A案/`

タスクの新規作成フォーム画面。タスク名・優先度・サブタスクを入力して登録する。
```

### 補足

- 1 行目は **日本語の画面名**（ファイル名と一致）
- メタ情報（`画面:` = 実装ファイルパス / `モック:` = 採用モックの配置パス）を 2 行目以降に併記

## `## 要素一覧`（詳細ファイル）

### 記述例

```markdown
## 要素一覧

| 要素 | 種類 | 位置 | 説明 | 表示条件 | 必須 | 制限 | 初期値 | 取得元 | アクション | 補足 |
| --- | --- | --- | --- | --- | --- | --- | --- | --- | --- | --- |
| タスク名入力欄 | input | フォーム本体 | タスク名を入力 | 常時 | ✅ | 100 文字以内 | 空（編集時は取得値） | `タスク詳細取得.task.title` | - | - |
| 優先度選択 | select | フォーム本体 | 優先度を選択 | 常時 | - | low / medium / high | medium（編集時は取得値） | `タスク詳細取得.task.priority` | - | - |
| 期限入力欄 | date | フォーム本体 | 期限日を指定 | 常時 | - | 当日以降 | 空（編集時は取得値） | `タスク詳細取得.task.dueDate` | - | - |
| サブタスク追加ボタン | button | フォーム本体 | サブタスク行を追加 | 常時 | - | 20 件まで | - | - | クリックでサブタスク入力行を追加 | - |
| 保存ボタン | button | フッター | 入力内容でタスクを作成 | 常時 | - | - | - | - | クリックで保存し一覧画面へ遷移 | 活性条件は ※1 |
| 削除ボタン | button | ヘッダー | タスクを削除 | 編集時のみ（新規作成時は非表示） | - | - | - | - | クリックで確認ダイアログを表示 | - |

※1: 必須項目（タスク名）が入力済み、かつ制限違反のフィールドが 1 つもない場合のみ活性。
```

### 補足

**要素 / 種類 / 位置列:**
- 要素はボタン・入力欄・ラベル・テーブル等の日本語論理名
- 種類は button / input / select / table / label 等
- 位置は ヘッダー / フッター / フォーム本体 / サイドバー 等

**表示条件 / 必須 / 制限 / 初期値列:**
- 表示条件は「常時」または出し分けの条件（ロール・状態・モード）を日本語で書く
- 制限は入力の上限・許容値・形式（`100 文字以内` / `low / medium / high` / `当日以降` 等）
- 初期値は画面表示時点の値（`空` / `medium` / 取得値 等）

**取得元列:**
- 表示・初期値の出所を `{バックエンド結合の論理名}.{フィールドパス}` 形式で書く（例: `タスク詳細取得.task.title`）
- 画面内で完結する要素（固定値・新規入力のみ・ボタン）は `-`

**アクション列:**
- 日本語のドメイン記述で書く。
  例: 「クリックで詳細画面へ遷移」「クリックで保存」

**補足:**
- 表に書き切れない複雑なロジックは `※1` `※2` … の印を該当セルに入れ、表の直下に `※1: 〜` 形式で詳細を記述
- 該当しない列は `-`
