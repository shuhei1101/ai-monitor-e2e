# ai-monitor テンプレート: 設定

プロジェクトの **設定値 / 環境変数** を集約する書式。
`settings.yaml` / `config.ini` / `.env` / `pydantic-settings` などで管理される値の一覧を、実装ファイルを開かなくても把握できる形にまとめる。

## 前提

- 設定値は **環境変数で上書きできる** ことをデフォルトとする
- 環境変数名は **キーパスを `UPPER_SNAKE_CASE` 化 + プロジェクト接頭辞** で機械的に決まる
  - プロジェクト接頭辞（`{PROJECT_PREFIX}`）は **プロジェクトごとに定める**（例: `MYAPP_` / `INGEST_` / `AITUBER_`）
  - 例: `database.host` → `{PROJECT_PREFIX}_DATABASE_HOST`
  - 例: `openai.model` → `{PROJECT_PREFIX}_OPENAI_MODEL`
- 読み込み優先度: **環境変数 > 設定ファイル > コードのデフォルト**（下ほど弱い）
- **秘匿情報**（API キー / DB パスワード等）は設定ファイルに書かず、環境変数専用で受ける

## 配置

- 常に **フォルダ + サブシステム別ファイル** で配置する（1 サブシステムしかなくても同じ）
  - 起点フォルダ: `設計図/設定/`
  - 各ファイル: `設計図/設定/{サブシステム名}.md`（例: `設計図/設定/バックエンド.md` / `設計図/設定/フロントエンド.md` / `設計図/設定/AI サーバー.md`）
- 1 ファイル運用は禁止（将来サブシステム追加時にファイル名変更が必要になり拡張性が悪いため）

## 担当セクション一覧

| No | セクション | サブセクション | 必須or条件 | 補足 |
| --- | --- | --- | --- | --- |
| 1 | `## 全設定一覧` | - | 必須 | 全キーを 1 表で一覧化 |
| 2 | `## 環境変数のみで受け取る値` | - | 秘匿情報 or 環境依存の切替がある場合 | パスワード / API キー / エンドポイント等 |
| 3 | `## 環境別デフォルト` | - | dev/stg/prod で差分がある場合 | 差分だけを書く（全キーを繰り返さない） |
| 4 | `## 補足` | - | 常に残す（なければ「なし」） | 命名規則の例外、廃止予定キー、移行注意点など |

## `冒頭リード`

### 記述例

```markdown
# 設定: {サブシステム名}

`{サブシステム名}` の設定値一覧。

- 環境変数で全キーを上書き可能。環境変数名は キーパスを UPPER_SNAKE_CASE 化 + プロジェクト接頭辞（本プロジェクトは `{PROJECT_PREFIX}_`）
- 秘匿情報は環境変数専用（本ファイルの 全設定一覧 から除外し、環境変数のみで受け取る値 に載せる）
- 実装は `settings.py` / `settings.yaml` / `.env` を参照
```

### 補足

- 環境変数接頭辞（`{PROJECT_PREFIX}_` 等）はプロジェクト方針に合わせて記載
- 実装ファイルへのリンクを冒頭に置くと開発者が飛びやすい

## `## 全設定一覧`

### 記述例

```markdown
## 全設定一覧

| No | 論理名 | キーパス | 型 | 既定値 | 環境変数名 | 秘匿 | 説明 | 補足 |
| --- | --- | --- | --- | --- | --- | --- | --- | --- |
| 1 | DB ホスト | `database.host` | `str` | `localhost` | `{PROJECT_PREFIX}_DATABASE_HOST` | - | 接続先ホスト | - |
| 2 | DB ポート | `database.port` | `int` | `5432` | `{PROJECT_PREFIX}_DATABASE_PORT` | - | 接続先ポート | - |
| 3 | ログレベル | `logging.level` | `str` | `INFO` | `{PROJECT_PREFIX}_LOGGING_LEVEL` | - | `DEBUG` / `INFO` / `WARN` | - |
| 4 | LLM モデル | `openai.model` | `str` | `gpt-4.1` | `{PROJECT_PREFIX}_OPENAI_MODEL` | - | 既定モデル | - |
| 5 | LLM タイムアウト | `openai.timeout_ms` | `int` | `30000` | `{PROJECT_PREFIX}_OPENAI_TIMEOUT_MS` | - | 単発リクエスト上限 | - |
| 6 | 対応環境 | `env` | `Literal` | `development` | `{PROJECT_PREFIX}_ENV` | - | `development` / `staging` / `production` | - |
```

### 補足

**カラム:**
`No / 論理名 / キーパス / 型 / 既定値 / 環境変数名 / 秘匿 / 説明 / 補足`（9 列）

**論理名列:**
- 業務用語（例: `DB ホスト` / `LLM モデル`）

**キーパス列:**
- 設定ファイルの階層構造をドットで結んだキー（例: `database.host`）
- フラットな設定なら 1 語（例: `env`）

**型列:**
- プロジェクト言語の型表記（`str` / `int` / `bool` / `Literal[...]` / `list[str]` 等）

**既定値列:**
- コード or 設定ファイルのデフォルト値
- 未設定を許容する場合は `None` / `-`

**環境変数名列:**
- 上書き用の環境変数名。原則機械的に決まる（キーパスを大文字化 + 接頭辞）
- 例外的な命名（既存互換など）は補足列に理由を書く

**秘匿列:**
- `✓` or `-`
- `✓` の場合は 全設定一覧 から外し、`## 環境変数のみで受け取る値` に移す（設定ファイルに書かない）

**補足:**
- 廃止予定 / 移行中のキーは補足列で明示（`廃止予定: v2.0 で削除`）

## `## 環境変数のみで受け取る値`

### 記述例

```markdown
## 環境変数のみで受け取る値

| No | 論理名 | 環境変数名 | 型 | 必須 | 説明 | 補足 |
| --- | --- | --- | --- | --- | --- | --- |
| 1 | DB パスワード | `{PROJECT_PREFIX}_DATABASE_PASSWORD` | `str` | ○ | DB 接続パスワード | 秘匿 |
| 2 | OpenAI API キー | `{PROJECT_PREFIX}_OPENAI_API_KEY` | `str` | ○ | OpenAI 認証 | 秘匿。ローテーションは 90 日ごと |
| 3 | GitHub Token | `{PROJECT_PREFIX}_GITHUB_TOKEN` | `str` | ○ | GitHub API 呼び出し用 | 秘匿 |
```

### 補足

**カラム:**
`No / 論理名 / 環境変数名 / 型 / 必須 / 説明 / 補足`（7 列）

**必須列:**
- `○` / `-`
- `○` で欠落時の挙動を補足列で明示（`起動時にエラー` / `該当機能を無効化` 等）

**補足:**
- ローテーション周期 / 発行元 / 秘匿レベル があれば補足列に

## `## 環境別デフォルト`

### 記述例

```markdown
## 環境別デフォルト

dev / stg / prod で差分がある値のみ列挙する。`## 全設定一覧` の既定値と異なるキーだけを書く。

| No | キーパス | development | staging | production | 補足 |
| --- | --- | --- | --- | --- | --- |
| 1 | `logging.level` | `DEBUG` | `INFO` | `WARN` | 本番はノイズを減らす |
| 2 | `openai.timeout_ms` | `10000` | `30000` | `30000` | dev は素早く失敗させる |
```

### 補足

- **差分がある値だけ書く**（全キーを繰り返さない）
- 環境自体は `env` キー（`development` / `staging` / `production`）で判定

## `## 補足`

### 記述例

```markdown
## 補足

- 環境変数接頭辞: `{PROJECT_PREFIX}_`（プロジェクト固有、実プロジェクト名で置換）
- 実装: `{path/to/settings.py 等}`（pydantic-settings 等）
- `.env.example` にサンプル値を置く
- 秘匿値のバリデーションは起動時 fail-fast
```

### 補足

- 命名の例外 / 実装ファイルへのリンク / 廃止予定 / 移行注意点をまとめる
- 特筆なければ「なし」と明記（省略しない）

## 表のルール（全テンプレ共通）

`docs/wiki/テンプレート/テンプレート.md` の「表のルール」に従う。
- No 列: 最左に必須
- 補足列: 最右に必須
- 連続同値の省略: `〃`
- 該当なし: `-`
