# ai-monitor テンプレート: テンプレート

`docs/wiki/テンプレート/{対象}.md` 形式のテンプレートページを作るときの共通書式（メタテンプレート）。
すべてのテンプレート Wiki は本ページの構造に従う。

**用語の使い分け（重要）:**
- **テンプレート** (`docs/wiki/テンプレート/*.md`) … 実体ファイル（イシュー本文 / PR 本文 / ER 図 / 外部 API 等）の **書式定義**。
  「何を / どう書くか」の雛形
- **規約** (`docs/wiki/規約/*.md`) … プロセス・手順・ルール。
  「どう作業するか」を定める（例: `docs/wiki/規約/モック画面.md`）
- 本ページは「テンプレートを書くための雛形」= **メタテンプレート** なので `docs/wiki/テンプレート/テンプレート.md`

**テンプレートには書式だけを書く（重要）:**
- テンプレートは本文の**たたき台**。
  書いてよいのは 冒頭リード（何の書式か 1〜3 行）/ セクション一覧 / 各セクションの記述例と記入時の補足 のみ
- **背景説明・運用フロー・検討の進め方・他方式との対比 は書かない**（テンプレートの責務ではない）
- テンプレートは他プロジェクトにも流用するため、**特定プロジェクト固有の内容を含ませない**
- 補足は**正の指示（何をどう書くか）だけ**にする。
  記述例に無い内容はそもそも出力されないので、「〜は書かない」式の禁止注釈を並べない（情報量ゼロの制約はコンテキストを汚し精度を下げる）

配置先は `docs/wiki/テンプレート/{対象}.md`（例: `docs/wiki/テンプレート/外部API.md`）。
「ページ」サフィックスは付けない。

## セクション一覧

| セクション | サブセクション | 必須or条件 | 担当 |
| --- | --- | --- | --- |
| 冒頭リード | - | 必須 | （規約整備者） |
| `## セクション一覧` | - | 必須 | （規約整備者） |
| `## `## {セクション名}`` | `### 記述例` / `### 補足` | 必須（実体ファイルの各 H2 セクションにつき 1 つ） | （規約整備者） |
| `## `### {サブセクション名}`` | `### 記述例` / `### 補足` | 動的サブセクションを定義する場合のみ | （規約整備者） |

## `冒頭リード`

### 記述例

```markdown
# ai-monitor テンプレート: {対象}

（このページが何を定義しているか・何のために存在するかを 1〜3 行）

配置先は `docs/wiki/{分類}/{ページ}.md`（例: `docs/wiki/外部API/OpenAI.md`）。
新規作成時の手順（例: 「同フォルダの `README.md`（一覧）に行追加」）も冒頭に書く。
```

### 補足

- ページの目的を最初に明示し、読み手が「何の規約か」を 3 秒で把握できるようにする
- 実体ファイルのファイル名規約・命名規則を冒頭で固定
- 新規作成時に行う付随作業（インデックスへの追記など）も冒頭でまとめる
- 横に長くなりすぎないように「。」で区切って改行する

## `セクション一覧`

### 記述例

```markdown
## セクション一覧

| セクション | サブセクション | 必須or条件 | 担当 |
| --- | --- | --- | --- |
| `## 概要` | - | 必須 | issue-arch / issue-ui |
| `## メソッド一覧` | `### {メソッド名}` | 必須（一覧表の各行につき 1 サブセクション） | issue-arch / issue-ui |
```

### 補足

**セクション列:**
- 実体ファイルに置く H2 セクション名を `## {名前}` 形式（バッククォート囲み）で

**サブセクション列:**
- 動的サブセクションがあれば `### {名前テンプレ}`、なければ `-`

**必須or条件列:**
- `必須` / `{条件} の場合のみ` / `必須（{追加条件}）` などで明示

**担当列:**
- その実体セクションを書く担当エージェント名（規約自体の更新者ではない）

**補足:**
- 冒頭リードの直後に必ず置く。
  読み手が全体構造を一望できるようにする
- 順序は実体ファイルでの登場順と揃える

## `## {セクション名}`

実体ファイルの各 H2 セクションについて、書き方を定義する単位。
1 セクションにつき 1 つの `## {名前}` 見出しを置く。

### 記述例

````markdown
## `## 概要`

**概要:**

このセクションが何を書く場所か・何のために書くかを 1〜2 文で。

**記述例:**

```markdown
## 概要

（実物のコピペ可能な Markdown 記入例）
```

**{カラム名}列:**
- 表があるセクションの場合、各列の固有ルールをここに書く
- 補足列は基本省略（特殊ルールがあれば書く）

**補足:**
- 全体的な運用ルール・注意点
- 表の場合は「表に書ききれない内容は `※n` で表外に詳細」など
````

### 補足

**順序:**
- **概要 → 記述例 → カラム別ルール → 補足** を厳守
- 「概要」は省略可（特に説明不要なシンプルなセクションのみ）

**記述例の書式:**
- ` ```markdown ` のコードフェンスで実物をそのまま貼る
- コードフェンスを入れ子にする場合は外側を ` ```` ` で 4 連バッククォート

**表のあるセクションのカラム別ルール:**
- `**{カラム名}列:**` の太字見出し + 箇条書きで各列のルール
- 特殊ルールがある列だけ書く
- 補足列は汎用のため通常省略

**補足ブロック:**
- 必ず最後に置く（カラム別ルールの後）
- 全体に共通する注意点・運用ルールを箇条書きで

## 文章のルール（全規約共通）

本文（冒頭リード・説明文・箇条書き）は**句点（。）ごとに改行する**（1 文 = 1 行。レンダリングでは 1 段落に結合される）。
エディタで横スクロールが発生する長い 1 行を作らない。
文を書き足すときも既存行への連結ではなく新しい行に足す。
表のセル・見出しは対象外（セル内の改行は `<br>`）。

## 表のルール（全規約共通）

すべての表に以下を必須化する。

| ルール | 内容 |
| --- | --- |
| 補足列 | 最右に必須（任意の追加情報用） |
| 表外の詳細 | 表に書ききれない複雑な内容は `※1` `※2` … の印を該当セルに入れ、表の直下に `※1: 〜` で記述 |
| 該当なし | 値が該当しない列は `-` |
