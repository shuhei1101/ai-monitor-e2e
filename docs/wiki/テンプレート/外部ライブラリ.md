# ai-monitor 規約: 外部ライブラリ

採用した外部ライブラリ 1 つにつき 1 ページの Wiki を作成し、**そのプロジェクトで実際に使う部分だけ** を公式ドキュメントから抽出して整理する。

公式ドキュメントは情報量が多く読みづらいため、ここを見るだけで実装に必要な情報（インストール手順 + 使うメソッドのパラメータ）が揃う状態を目指す。

配置先は `docs/wiki/外部ライブラリ/{ライブラリ名}.md`（例: `docs/wiki/外部ライブラリ/react-image-crop.md`）。
新規作成時は `docs/wiki/外部ライブラリ/README.md`（一覧）に行を追加してリンクを張る。

参照方向は **モジュール構成 → 外部ライブラリ の一方向**。
本ページは依存の最下層として当該ライブラリの情報だけを書き、自プロジェクトのファイル・モジュールには言及しない。

各セクションの書き方を「記述例 + 補足」で定義する。

## セクション一覧

| セクション | サブセクション | 必須or条件 | 担当 | 補足 |
| --- | --- | --- | --- | --- |
| `## 概要` | - | 必須 | issue-arch / issue-ui | - |
| `## 現在のバージョン情報` | - | 必須 | issue-arch / issue-ui | - |
| `## インストール手順` | - | 必須 | issue-arch / issue-ui | - |
| `## メソッド一覧` | `### {メソッド名 / コンポーネント名}` | 必須（一覧表の各行につき 1 サブセクション） | issue-arch / issue-ui | - |

## `## 概要`

### 記述例

```markdown
## 概要

React 用の画像トリミング（クロップ）UI ライブラリ。

矩形・円形・自由比率のトリミング選択と、選択範囲のリアルタイムプレビューを提供する。
出力はトリミング後の Blob / dataURL。
```

### 補足

- ライブラリが何をするものかを 1〜3 行で要約する
- 自プロジェクトでの用途・採用箇所は書かない

## `## 現在のバージョン情報`

### 記述例

```markdown
## 現在のバージョン情報

| 項目 | 内容 |
| --- | --- |
| バージョン | `v11.0.5` |
| ライセンス | MIT |
| 公式 URL | https://github.com/DominicTobias/react-image-crop |
| 公式ドキュメント | https://github.com/DominicTobias/react-image-crop#readme |
```

### 補足

- バージョンは採用時点で固定値を記録。
  アップグレードしたらここを更新（メソッド・パラメータも追従して更新する）
- ライセンスは商用利用可否を確認した上で記入（MIT / Apache-2.0 / GPL など）
- 公式 URL は GitHub / npm / 公式サイトのいずれか主要なもの 1 つ
- 公式ドキュメントは API リファレンスがある URL を直接

## `## インストール手順`

### 記述例

````markdown
## インストール手順

```bash
npm install react-image-crop@11.0.5
```

CSS の読み込みも必要:

```ts
import "react-image-crop/dist/ReactCrop.css";

````

### 補足

- パッケージマネージャに応じた具体コマンド（npm / pnpm / yarn / uv / pip など）
- バージョンを `@x.y.z` で固定指定
- ピア依存・追加設定（CSS インポート / Tailwind 設定 / 型定義 / 環境変数 など）があれば併記

## `## メソッド一覧`

このプロジェクトで使うメソッド / コンポーネントを 1 表で俯瞰し、表の下に各メソッドの詳細サブセクション（`### {名前}`）を並べる。

### 記述例

````markdown

バージョン: `v11.0.5`

| 種別 | 名前 | 用途 | 補足 |
| --- | --- | --- | --- |
| コンポーネント | [`<ReactCrop />`](#reactcrop) | クロップ UI を表示するメインコンポーネント | - |
| 関数 | [`centerCrop()`](#centercrop) | クロップ範囲を画像中央に配置 | - |

### `ReactCrop`

クロップ UI を表示するメインコンポーネント。

#### パラメータ

| パラメータ | 型 | 必須 | 既定 | 説明 | 補足 |
| --- | --- | --- | --- | --- | --- |
| `crop` | `Crop` | 任意 | `undefined` | 現在のクロップ範囲（controlled 利用時） | 子フィールドは下の行で展開 |
| `crop.unit` | `'%' or 'px'` | 必須 | - | 座標・サイズの単位 | `%`=画像幅比 / `px`=画素絶対値 |
| `crop.x` | `number` | 必須 | - | 左上 X 座標 | - |
| `crop.y` | `number` | 必須 | - | 左上 Y 座標 | - |
| `crop.width` | `number` | 必須 | - | 幅 | - |
| `crop.height` | `number` | 必須 | - | 高さ | - |
| `onChange` | `(crop: PixelCrop, percentCrop: Crop) => void` | 必須 | - | クロップ範囲変更時のコールバック | - |
| `onComplete` | `(crop: PixelCrop) => void` | 任意 | `undefined` | ドラッグ終了時の確定コールバック | プレビュー画像生成用 |
| `aspect` | `number` | 任意 | `undefined` | アスペクト比固定 | `1`=正方形 / `16/9`=横長 / 未指定=自由比率 |
| `circularCrop` | `boolean` | 任意 | `false` | 円形クロップ表示 | 出力データ自体は矩形 |
| `keepSelection` | `boolean` | 任意 | `false` | クリック時に選択範囲を保持 | - |
| `disabled` | `boolean` | 任意 | `false` | クロップ操作を無効化 | - |
| `minWidth` | `number` | 任意 | `0` | 最小幅 (px) | - |
| `maxWidth` | `number` | 任意 | `undefined` | 最大幅 (px) | - |

パラメータ例:

```tsx

```

| パラメータ | 型 | 必須 | 既定 | 説明 | 補足 |
| --- | --- | --- | --- | --- | --- |
| `crop` | `Crop` | 必須 | - | 中央配置したいクロップ範囲 | 子フィールドは下の行で展開 |
````

### 補足

**一覧表の種別列:**
- `コンポーネント` / `関数` / `フック` / `クラス` / `定数` / `コマンド`（CLI ツールの場合）など

**一覧表の名前列:**
- 詳細サブセクションへのリンクで書く（`[\`名前\`](#{アンカー})`。アンカー = 見出しを小文字化 + 空白を `-` に置換 + 記号除去）

**一覧表の用途列:**
- 1 行で要約。
  詳細パラメータは下の `### {名前}` サブセクション側で書く

**各メソッドサブセクション:**
- 一覧表の各行につき 1 つ作成。
  見出しは `### \`{メソッド名}\``（引数はパラメータ表側で書く）
- 構成: 1〜2 行の説明 → `#### パラメータ`（表 + `パラメータ例:`）→ `#### 戻り値`（表 + `戻り値例:`）
- パラメータ表カラム: `パラメータ` / `型` / `必須` / `既定` / `説明` / `補足` の 6 列固定
- 戻り値表カラム: 単純な値は `型` / `説明` / `補足` の 3 列。
  オブジェクト / JSON を返す場合は `フィールド / 型 / 説明 / 補足` の 4 列で 1 フィールド = 1 行に展開する（パス表記はパラメータ表の入れ子ルールと同じ。返らない / `null` があり得るフィールドは `フィールド?` と表記し、条件を補足列に書く）
- 戻り値が自明なコンポーネントなどは `#### 戻り値` を省略可
- `パラメータ例:` は import 文込みの最小動作版（5〜15 行程度）・`戻り値例:` は返り値の実形（JSON / オブジェクトリテラル）

**パラメータの網羅性:**
- 「現在使っているパラメータ」だけでなく **主要パラメータも全部** 載せる
- 理由: 採用後にチューニングで別パラメータを足す可能性が高く、その都度公式ドキュメントを引き直すのを避けたい
- 採用判断時に「他にどんなパラメータがあるか」が分かる状態にしておく
- 公式ドキュメント全列挙ではなく **主要 = 通常使われる範囲** に絞る（明らかに使わない upstream-only / deprecated は除外）

**「既定」列の書き方:**
- 任意パラメータの省略時デフォルト値を書く（例: `false` / `0` / `undefined` / `'auto'`）
- 必須パラメータは `-` 固定
- 公式が明示してない場合は `undefined` または `-`

**enum パラメータ（値で動作が変わるもの）の書き方:**
- 型列: `'a' or 'b' or 'c'` で取りうる値を列挙
- 補足列: 各値の意味を `a=～ / b=～ / c=～` 形式で書く
- 値が多い（5 個以上など）ときのみ別表 / 別サブセクションに切り出してよい

**データクラス型パラメータ（オブジェクト 1 つを受け取るもの）の書き方:**
- ライブラリは型エイリアスやデータクラス（`Crop` / `Options` など）でパラメータをまとめて渡すケースが多い
- **行を追加して展開する**（API 側の入れ子ルールと同じ）
  - 親行: 型を `Crop` で書き、補足列に「子フィールドは下の行で展開」
  - 子行: `crop.unit` / `crop.x` のように `parent.child` 形式で各フィールドを 1 行ずつ
- **展開は 1 段目まで**（読みやすさ優先）
  - 子フィールドの型がさらに別クラスの場合、孫フィールドは展開しない
  - 孫の構造は子行の補足列に型定義 `Sub = { ... }` で書く
- フィールドが多い（10 個以上） / 同じ型を複数メソッドで使う場合は、別サブセクション `### {型名} 型` に切り出して各メソッドからは「`Crop` 型は下の `### Crop 型` 参照」で済ます

**入れ子（オブジェクト / 配列の中身）の書き方:**
- パラメータ列の表記:
  - オブジェクトの子: `parent.child`
  - 配列要素の子: `parent[].child`
- 親行の補足列に「子フィールドは下の行で展開」と書く

**補足:**
- **メソッド単位では** このプロジェクトで実際に使うものだけ抜粋
- 採用メソッドのパラメータは網羅的に書く（上記「パラメータの網羅性」参照）
- 表の上に **採用しているバージョン** を必ず明記。
  アップグレード時の見直しトリガーになる
- 表とサブセクションの順序を揃える（使用頻度の高い順 or 依存順）
