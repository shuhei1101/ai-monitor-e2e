# ai-monitor テンプレート: スキル

エージェントの SKILL.md（`plugins/ai-monitor/skills/{エージェント名}/SKILL.md`）の書式。
スキル本文には作業手順・判定基準を書かず、Wiki（フェーズ索引 + フェーズページ）を動的注入 / 実行時取得で参照する。
Wiki 側の push だけで挙動が更新され、マーケットプレイスの更新を伴わない。

## ファイル構成

| 種類 | ファイル | 役割 | 補足 |
| --- | --- | --- | --- |
| スキル | `plugins/ai-monitor/skills/{エージェント名}/SKILL.md` | エージェントの起動入口（入力 + 初期処理 + 参照する Wiki の列挙） | 1 エージェント = 1 スキル |

## セクション一覧

| セクション | 必須or条件 | 補足 |
| --- | --- | --- |
| フロントマター | 必須 | `name` / `description` / `argument-hint` / `arguments` / `disable-model-invocation`（モデル指定時は `model` も） |
| 冒頭リード | 必須 | `# {エージェント名}` + 役割 1〜2 行 |
| `## 入力` | 必須 | 引数の一覧 |
| `## フェーズ` | 必須 | フェーズ索引 + 全フェーズページの動的注入を上から並べる |
| `## 参考資料` | 必須 | 動的注入する Wiki ページを上から順に並べる（末尾に置く） |

## `フロントマター`

### 記述例

```markdown
---
name: ai-monitor:story-conductor
description: story レイヤーの指揮役
argument-hint: "[issue-number]"
arguments: "issue_number"
disable-model-invocation: true
---
```

### 補足

- `name` は `ai-monitor:{エージェント名}`（エージェント名 = スキルフォルダ名 = tmux セッション名の一部）
- `description` は役割の 1 行のみ（自動起動させないため、モデルに読ませるトリガー説明は不要）
- `disable-model-invocation: true` を必ず付ける（起動はモニターの send-keys による `/ai-monitor:{エージェント名}` 明示呼び出しのみ。モデルの自動判断では呼ばせない）
- 使用モデルを指定する場合は `model:` を追加する（値の SoT はこのフロントマター）

## `冒頭リード`

### 記述例

```markdown
# story-conductor

story レイヤーの指揮役。
1 ユースケースの要件を確定し、実装分担（subsystem）への分解から epic への昇格マージまでを進行させる。
```

### 補足

- 役割を概要レベルの抽象度で 1〜2 行だけ書く（フェーズ名・工程の列挙はしない）

## `## 入力`

### 記述例

```markdown
## 入力

- Issue 番号: $issue_number
```

### 補足

- フロントマター `arguments` で宣言した位置引数を論理名付きで列挙する

## `## フェーズ`

### 記述例

````markdown
## フェーズ

!`python "${CLAUDE_PLUGIN_ROOT}/inject/read_urls.py" "${WIKI_BASE}/エージェント/story-conductor/README.md"`

!`python "${CLAUDE_PLUGIN_ROOT}/inject/read_urls.py" "${WIKI_BASE}/エージェント/story-conductor/フェーズ/初期処理.md"`

!`python "${CLAUDE_PLUGIN_ROOT}/inject/read_urls.py" "${WIKI_BASE}/エージェント/story-conductor/フェーズ/要件確定（初回）.md"`

!`python "${CLAUDE_PLUGIN_ROOT}/inject/read_urls.py" "${WIKI_BASE}/エージェント/story-conductor/フェーズ/要件確定（応答ループ）.md"`
````

### 補足

- フェーズ索引 + 全フェーズページをスキル読み込み時の動的注入で展開する（注入ごとの見出しは作らない。各ページが自身の見出しを持つ）
- 先頭にフェーズ索引（`エージェント/{エージェント名}/README.md`。起動条件の判定表）、次に 初期処理、以降はフェーズ索引のフェーズ一覧と同じ並びで置く

## `## 参考資料`

### 記述例

````markdown
## 参考資料

!`python "${CLAUDE_PLUGIN_ROOT}/inject/read_agent_docs.py" story-conductor`
````

### 補足

- スキルの末尾に置き、動的注入（`!` 記法はスキル読み込み時に展開される）を上から順に並べる
- 注入ごとの見出しは作らない（各ページが自身の見出しを持つ）
- `read_agent_docs.py` は共通ルール（`Claudeハーネス/共通ルール/`）と、対応表（`共通対応表/` = 全プロジェクト共通 / `対応表/` = プロジェクト固有）で自分に ○ の付いたドキュメントを一括展開する。
  参照ドキュメントの増減はスキルではなく対応表側で行う
- 常時必要なページだけをここに置く。
  特定フェーズでしか使わないページはフェーズページ側から実行時取得する
