# ai-monitor テンプレート: PR 本文 / サブシステム

`layer:subsystem` が付いた PR の本文書式。

subsystem PR は **実装 + 設計図 Wiki 更新のための PR**。
変更内容の詳細（SS 設計 / UI 設計 / 実装計画 / 追記した Wiki 一覧 等）は**書かない**（変更ファイル一覧に落ちるので二重管理を避ける）。
書くのは **タスク一覧（実施すべき To Do）** と **実行したテストの結果** だけ。

## セクション一覧

| セクション | サブセクション | 必須or条件 | 補足 |
| --- | --- | --- | --- |
| `## 紐づく Issue` | - | 必須 | 親 subsystem Issue 番号 |
| `## タスク一覧` | - | 必須 | Wiki 修正・実装・テスト実行の To Do |
| `## 単体テスト結果` | - | 必須 | 全単体テストを実行して結果を記録 |
| `## 結合テスト結果` | - | 必須 | 関連する結合テストのみ実行して結果を記録 |
| `## 単一ユースケースシナリオテスト結果` | - | 必須 | 関連する単一 UC E2E テストのみ実行して結果を記録 |
| `## 複合ユースケースシナリオテスト結果` | - | 必須 | 関連する複合 UC E2E テストのみ実行して結果を記録 |

## `## 紐づく Issue`

### 記述例

```markdown
## 紐づく Issue

- #200 プロフィール編集 BE サブシステム
```

### 補足

- 親 subsystem Issue 番号を必須で書く（1 件のみ。祖先は Sub-issue リンクで辿れる）

## `## タスク一覧`

### 記述例

```markdown
## タスク一覧

- [ ] `設計図/モジュール構成/ユーザー.md` を更新
- [ ] `設計図/バックエンド結合/ユーザー編集.md` を新規作成
- [ ] `設計図/ER図/ユーザー.md` の `users` テーブルに `bio` カラムを追記
- [ ] `外部ライブラリ/react-image-crop.md` を新規作成
- [ ] 実装コードを書く
- [ ] 単体テストを追加 + 実行
- [ ] 関連する結合テストを実行
- [ ] 関連する単一 UC E2E テストを実行
- [ ] 関連する複合 UC E2E テストを実行
```

### 補足

- チェックボックス形式で **やることリスト**を列挙
- 各タスクは 1 行 = 1 To Do
- Wiki 修正・実装・テスト実行の 3 種類に分けて書く
- 完了項目に `[x]` を入れて、全てチェックしてからマージ

## `## 単体テスト結果`

### 記述例

```markdown
## 単体テスト結果

| ファイル | メソッド | 結果 | 補足 |
| --- | --- | --- | --- |
| `tests/services/test_user_service.py` | 全実行 | ✅ | 3 テスト pass |
| `tests/repositories/test_user_repo.py` | 全実行 | ✅ | 2 テスト pass |
```

### 補足

- **単体テストは全部実行**して結果を記録する
- ファイル単位で 1 行、`メソッド` 列は `全実行` と書く（個別実行時のみメソッド名）
- 結果列: `✅`（pass）/ `❌`（fail）
- fail した場合は補足列にエラー要約
- 単体テストが 1 件もないプロジェクトなら本文に「なし」と記載

## `## 結合テスト結果`

### 記述例

```markdown
## 結合テスト結果

| ファイル | メソッド | 結果 | 補足 |
| --- | --- | --- | --- |
| `tests/api/test_users.py` | `test_put_users_id_e2e` | ✅ | 新規追加テスト |
| `tests/api/test_profile.py` | 全実行 | ✅ | 既存テスト、回帰確認 |
```

### 補足

- **関連する結合テストのみ実行**（今回の変更に該当するもの + 新規作成したもの）
- 全部回す必要はない
- 結果列: `✅`（pass）/ `❌`（fail）
- 関連する結合テストがない場合は本文に「なし」と記載

## `## 単一ユースケースシナリオテスト結果`

### 記述例

```markdown
## 単一ユースケースシナリオテスト結果

| ファイル | メソッド | 結果 | 補足 |
| --- | --- | --- | --- |
| `tests/e2e/profile_edit.spec.ts` | 全実行 | ✅ | 対応シナリオ: `プロフィール編集` |
```

### 補足

- **関連する単一 UC E2E テストのみ実行**
- 対応する `docs/wiki/設計図/シナリオ/単一ユースケース/{機能名}.md` のシナリオを満たしているか
- 結果列: `✅` / `❌`
- 補足列に対応シナリオ名を明記
- 関連する単一 UC テストがない場合は本文に「なし」と記載

## `## 複合ユースケースシナリオテスト結果`

### 記述例

```markdown
## 複合ユースケースシナリオテスト結果

| ファイル | メソッド | 結果 | 補足 |
| --- | --- | --- | --- |
| `tests/e2e/onboarding/signup-to-first-task.spec.ts` | 全実行 | ✅ | 対応シナリオ: `新規登録から初回タスク作成` |
```

### 補足

- **関連する複合 UC E2E テストのみ実行**
- 対応する `docs/wiki/設計図/シナリオ/複合ユースケース/{論理名}.md` のシナリオを満たしているか
- 結果列: `✅` / `❌`
- 補足列に対応シナリオ名を明記
- 関連する複合 UC テストがない場合は本文に「なし」と記載
