# ai-monitor テンプレート: PR 本文 / ストーリー

`layer:story` が付いた PR の本文書式。

story PR は **単一ユースケースシナリオを設計するための PR**。
親 story Issue の `## ユースケース要件` を元に、`docs/wiki/設計図/シナリオ/単一ユースケース/{機能名}.md` を作成 / 更新する。
書くのは **タスク一覧（実施すべき To Do）** と **単一ユースケースシナリオテストの結果** だけ（変更内容の詳細は変更ファイル一覧で見る）。

## セクション一覧

| セクション | サブセクション | 必須or条件 | 補足 |
| --- | --- | --- | --- |
| `## 紐づく Issue` | - | 必須 | 親 story Issue 番号 |
| `## タスク一覧` | - | 必須 | Wiki 修正・単一ユースケースシナリオテスト実行の To Do |
| `## 単一ユースケースシナリオテスト結果` | - | 必須 | 作成 / 変更した単一 UC E2E テストの結果 |

## `## 紐づく Issue`

### 記述例

```markdown
## 紐づく Issue

- #123 プロフィール編集ストーリー
```

### 補足

- 親 story Issue 番号を必須で書く（1 件のみ。祖先は Sub-issue リンクで辿れる）

## `## タスク一覧`

### 記述例

```markdown
## タスク一覧

- [ ] `設計図/シナリオ/単一ユースケース/プロフィール編集.md` を新規作成
- [ ] `設計図/シナリオ/README.md` の `## 一覧` に該当行を追加
- [ ] 単一ユースケース E2E テストを実行
```

### 補足

- チェックボックス形式で **やることリスト**を列挙
- 各タスクは 1 行 = 1 To Do
- Wiki 修正・テスト実行の 2 種類に分けて書く
- 完了項目に `[x]` を入れて、全てチェックしてからマージ

## `## 単一ユースケースシナリオテスト結果`

### 記述例

```markdown
## 単一ユースケースシナリオテスト結果

| ファイル | メソッド | 結果 | 補足 |
| --- | --- | --- | --- |
| `tests/e2e/profile_edit.spec.ts` | 全実行 | ✅ | 対応シナリオ: `プロフィール編集` |
```

### 補足

- 本 story で作成 / 変更した単一 UC E2E テストを全て実行して結果を記録
- 結果列: `✅`（pass）/ `❌`（fail）
- 補足列に対応シナリオ名を明記（`docs/wiki/設計図/シナリオ/単一ユースケース/{機能名}.md` と 1:1 対応）
- 単一ユースケースシナリオが無い story はそもそも成立しないが、テスト未整備の暫定 PR なら本文に「なし」と記載
