# ai-monitor テンプレート: 定数

プロジェクトの **業務ロジック上の定数**（設定値ではない不変値）を集約する書式。
`エンドポイント URL` / `テーマカラー` / `固定辞書` / `列挙値と一緒に使う対応表` などの、コード上グローバルに参照される即値をここに載せる。

## モジュール構成 との使い分け

- 定数 1 個の名前 / 型 / コンテナ を **他要素と並べて把握したい** → `モジュール構成/{分類}.md` の一覧に 種別 `定数` として書く（基本はこちら）
- 定数の **中身（辞書のキー / 値、配列の各要素）を一覧化して見せたい** → 本テンプレを使う
- 併用可: モジュール構成の一覧行に載せた上で、本ページに詳細を書く（相互リンク）

## 設定 との使い分け

- 環境変数で上書きされる値（`DB_HOST` / `LOG_LEVEL` 等）→ `テンプレート/設定.md`
- 実行時に変わらない業務ロジック上の即値（`エンドポイント一覧` / `テーマトークン` 等）→ 本テンプレ

## 配置

- 常に **フォルダ + サブシステム別ファイル** で配置する（1 サブシステムしかなくても同じ）
  - 起点フォルダ: `設計図/定数/`
  - 各ファイル: `設計図/定数/{サブシステム名}.md`（例: `設計図/定数/バックエンド.md` / `設計図/定数/フロントエンド.md`）
- 1 ファイル運用は禁止（将来サブシステム追加時にファイル名変更が必要になり拡張性が悪いため）
- サブシステム別に持つ定数の例:
  - `バックエンド`: API エンドポイント / ラベル辞書 / 業務コード
  - `フロントエンド`: UI テーマトークン / ルーティング定数 / 表示ラベル

## 担当セクション一覧

| No | セクション | サブセクション | 必須or条件 | 補足 |
| --- | --- | --- | --- | --- |
| 1 | `## 一覧` | - | 必須 | 全定数の早見表 |
| 2 | `## {定数名}` | `### 値` | 辞書 / 配列など内容が大きい定数のみ | 複雑な定数の中身を一覧化 |
| 3 | `## 補足` | - | 常に残す（なければ「なし」） | 命名規則の例外 / 廃止予定 / 参照元など |

## `冒頭リード`

### 記述例

```markdown
# 定数: {サブシステム名}

`{サブシステム名}` で使う業務ロジック上の定数一覧。

- 環境変数で上書きされる値は本ページではなく `設計図/設定.md` を参照
- 実装は `constants.ts` / `constants.py` / `theme.ts` などを参照
```

## `## 一覧`

### 記述例

```markdown
## 一覧

| No | 用途 | 種別 | 名前 | コンテナ | 型 / 例 | 補足 |
| --- | --- | --- | --- | --- | --- | --- |
| 1 | API エンドポイント | 定数辞書 | `API_ENDPOINTS` | `shared/constants.ts` | `Record<string, string>` → `/api/v1/users` 等 | - |
| 2 | 対応プロバイダ | 定数配列 | `SUPPORTED_PROVIDERS` | `oauth/constants.ts` | `readonly Provider[]` → `["google", "github", "apple"]` | - |
| 3 | 最大リトライ回数 | 定数 | `MAX_RETRY` | `shared/constants.ts` | `int = 3` | - |
| 4 | テーマトークン | 定数辞書 | `THEME_TOKENS` | `shared/theme.ts` | `Record<string, string>` → `--color-primary: #1976d2` 等 | - |
| 5 | 表示ラベル辞書 | 定数辞書 | `PROVIDER_LABELS` | `oauth/constants.ts` | `Record<Provider, string>` | - |
```

### 補足

**カラム:**
`No / 用途 / 種別 / 名前 / コンテナ / 型 / 例 / 補足`（7 列）

**種別列:**
- `定数`（単純即値: 数値 / 文字列 / 真偽値）
- `定数辞書`（キー→値の対応表: dict / Map / Record）
- `定数配列`（順序ありのリスト: list / tuple / readonly array）

**用途列:**
- どんな場面で使う定数か短く 1 語（`API エンドポイント` / `テーマトークン` / `表示ラベル辞書` 等）

**型 / 例列:**
- 型定義 + 代表的な値の抜粋
- 中身が大きい辞書 / 配列は例のみ、詳細は `## {定数名}` の `### 値` 表に

**補足:**
- 並び順は 用途 → 種別 → 名前
- 単純即値は 詳細 H3 セクションを立てない。辞書 / 配列で中身の一覧化が必要なもののみ H3 化

## `## {定数名}`

`### 値` サブセクションで内容を一覧化する。単純即値は H3 セクション自体を立てない。

### 記述例

````markdown
## `API_ENDPOINTS`

各バックエンド API のエンドポイントパス辞書。呼び出し側は文字列を直接書かず、必ずこの辞書を経由する。

### 値

| No | キー | 値 | 説明 | 補足 |
| --- | --- | --- | --- | --- |
| 1 | `USERS_LIST` | `/api/v1/users` | ユーザー一覧取得 | GET |
| 2 | `USERS_DETAIL` | `/api/v1/users/:id` | ユーザー詳細取得 | GET |
| 3 | `USERS_CREATE` | `/api/v1/users` | ユーザー新規作成 | POST |
| 4 | `AUTH_SIGNIN` | `/api/v1/auth/signin` | サインイン | POST |
| 5 | `AUTH_SIGNOUT` | `/api/v1/auth/signout` | サインアウト | POST |
````

### 補足

**`### 値` 表のカラム:**
`No / キー / 値 / 説明 / 補足`（5 列）

- 配列型の定数は `キー` 列を `index` に読み替える
- 動的計算される値（実行時に環境変数などから合成）は `-` にして `補足` 列で計算方法を明示

## `## 補足`

### 記述例

```markdown
## 補足

- 実装: `src/ai_monitor/shared/constants.py`
- テーマトークンは CSS カスタムプロパティで定義（`shared/theme.css`）
- エンドポイント文字列を直接書くのは禁止（必ず本辞書経由）
```

### 補足

- 実装ファイルへのリンク
- 使用ルール（「直接文字列で書かない」等）
- 廃止予定 / 移行注意点
- 特筆なければ「なし」（省略しない）

## 表のルール（全テンプレ共通）

`docs/wiki/テンプレート/テンプレート.md` の「表のルール」に従う。
- No 列: 最左に必須
- 補足列: 最右に必須
- 連続同値の省略: `〃`
- 該当なし: `-`
