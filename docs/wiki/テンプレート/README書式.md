# ai-monitor テンプレート: README

`docs/wiki/` 配下の各 `README.md`（ナビゲーション用ページ）の書式定義。
ルート（`docs/wiki/README.md`）も各階層（`docs/wiki/{カテゴリ}/.../README.md`）も同じ書式。

各 README は **自分のフォルダ直下の目次** だけを載せる。
深い階層は辿って見る方式。

ファイル名は `README.md` 固定。
配置は各ディレクトリ直下（1 ディレクトリ 1 ファイル）。

## セクション一覧

| セクション | サブセクション | 必須or条件 | 担当 | 補足 |
| --- | --- | --- | --- | --- |
| `# {カテゴリ名}` | - | 必須 | （Wiki 整備者） | 見出しのみ |
| `## 目次` | - | 必須 | （Wiki 整備者） | 当該ディレクトリ直下の一覧 |

## `# {カテゴリ名}`

### 記述例

```markdown
# テンプレート
```

### 補足

- フォルダ名と同じ日本語名（例: `# Wiki` / `# ai-monitor` / `# テンプレート` / `# 規約`）
- 概要・説明文は付けない（見出しのみ）

## `## 目次`

### 記述例

```markdown
## 目次

| ページ | 概要 |
| --- | --- |
| [{表示名}](./{ファイル名}.md) | {1 行説明} |
| [{サブフォルダ名}](./{サブフォルダ}/) | {1 行説明} |
```

### 補足

**対象:**
- 当該ディレクトリ **直下** のファイル + サブディレクトリ一覧
- 深い階層の中身は載せない（サブフォルダの README を辿って見る方式）

**カラム:**
- 最小は `ページ / 概要`
- カテゴリに応じて列追加可:
  - **テンプレ系**: `ページ / 対象実体ファイル / 概要`
  - **API 一覧系**: `ページ / 使用箇所 / 補足` 等

**ページ列:**
- ファイル: `[表示名](./{ファイル名}.md)` 形式
- サブディレクトリ: `[表示名](./{サブフォルダ}/)` 形式（末尾スラッシュで README.md が自動表示される）
- 表示名はファイル名 / フォルダ名そのまま（`.md` 抜きで OK）

**並び順:**
- 利用頻度 or 重要度の高い順
- 同種別のもの（ファイル群 / サブフォルダ群）を連続して並べる

**補足:**
- 「直下に何があるか」を 1 画面で把握できることが目的
