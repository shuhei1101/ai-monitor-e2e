# ai-monitor テンプレート: テスト実行方法

プロジェクトの **テスト種別ごとの実行コマンド** をまとめる書式。
トリアージ時にサブエージェントが「実際にテストを走らせる」ときの参照元として使う。

ファイル名は `テスト実行方法.md`（固定）。

## ファイル構成

`docs/wiki/テスト/テスト実行方法.md`

## `冒頭リード`

### 記述例

```markdown
# テスト実行方法

プロジェクトのテスト種別ごとの実行コマンドと前提条件。

## 前提

- Node.js 22 以上 + pnpm 9 以上
- DB: PostgreSQL 16 のローカル起動（`docker compose up -d postgres`）
- 環境変数: `.env.test` を配置
```

### 補足

- 全種別共通の前提条件（言語ランタイム / 依存サービス / 環境変数）を冒頭にまとめる
- 種別固有の前提は各セクションに書く

## `## E2E テスト`

### 記述例

````markdown
## E2E テスト

Playwright + Chromium ヘッドレスで実行。

**前提:**

- **フロントサーバー**: Playwright が `webServer` 設定で自動起動（手動不要）
- **バックエンドサーバー**: 別ターミナルで起動が必要
  ```bash
  uv run fastapi dev src/main.py --port 8000
  ```
- **DB 起動**: `docker compose up -d postgres-test`
- **外部 API のモック化**: デフォルトで Mock（MSW / pytest-httpx）
  - 実 API を叩きたい場合は `E2E_USE_REAL_EXTERNAL_API=true` 環境変数を付与（料金 / 副作用が発生するため要注意 → `## 外部疎通テスト` 参照）

**全件実行:**

```bash
pnpm test:e2e
```

**個別シナリオ実行:**

```bash
pnpm test:e2e {シナリオファイルパス}
```
````

### 補足（全種別共通）

- `**前提:**` を先頭に置く（種別固有の前提: 開発サーバー起動要否 / Mock 状況 / DB 起動 等）
- `**全件実行:**` / `**個別{X}実行:**` は各 1 コマンドだけコードブロックで書く
- 個別実行は **プレースホルダー形式**（`{シナリオファイルパス}` / `{画面テストファイルパス}` 等）で 1 コマンドだけ書く。
- コマンドは言語指定 `bash` で書く → GitHub の 1 クリックコピーで貼り付け可能

## `## 結合テスト（フロント）`

### 記述例

````markdown
## 結合テスト（フロント）

vitest + React Testing Library + MSW で実行。

**前提:**

- MSW でバックエンド API を Mock 化（実 API は叩かない）

**全件実行:**

```bash
pnpm test:integration:front
```

**個別画面実行:**

```bash
pnpm test:integration:front {画面テストファイルパス}
```
````

### 補足

- コマンドはリポジトリの `package.json` scripts と一致させる
- 記法ルールは E2E テストの補足を参照

## `## 結合テスト（バック）`

### 記述例

````markdown
## 結合テスト（バック）

pytest + httpx TestClient で実行。DB は Docker で立ち上げたテスト用インスタンス。

**前提:**

- DB 起動: `docker compose up -d postgres-test`
- マイグレーション: `uv run alembic upgrade head`
- 外部 API は `pytest-httpx` などで Mock 化（実 API は叩かない）

**全件実行:**

```bash
uv run pytest tests/integration/
```

**個別エンドポイント実行:**

```bash
uv run pytest {エンドポイントテストファイルパス}
```
````

### 補足

- DB / 外部依存の起動手順を前提に明記
- 記法ルールは E2E テストの補足を参照

## `## 外部疎通テスト`

### 記述例

````markdown
## 外部疎通テスト

**実 API に接続して疎通を確認** するテスト。
料金発生 / 副作用（実メール送信 / 実決済 / 実 SMS 等）があるため **通常の CI では実行しない**。

**前提:**

- 実 API 認証情報を環境変数 or `.env.external` に設定
  - 例: `OPENAI_API_KEY` / `SENDGRID_API_KEY` / `STRIPE_SECRET_KEY`
- **料金 / 副作用の発生を認識した上で実行**（テストごとの料金目安は `外部API/{API名}.md` を参照）
- 誤実行防止のため `--run-external` フラグ必須（フラグなしだと skip される）
- 通常の CI（GitHub Actions 等）では **実行禁止**

**全件実行:**

```bash
uv run pytest tests/external/ --run-external
```

**個別実行:**

```bash
uv run pytest {外部疎通テストファイルパス} --run-external
```
````

### 補足

- 料金 / 副作用の警告を **必ず前提に明記**
- `--run-external` のような誤実行防止フラグを必須にする
- 対象外部 API の詳細は `外部API/{API名}.md` を参照させる
- 記法ルールは E2E テストの補足を参照
