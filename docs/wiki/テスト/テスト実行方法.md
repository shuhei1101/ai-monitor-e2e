# テスト実行方法

プロジェクトのテスト種別ごとの実行コマンドと前提条件。

## 前提

- Python 3.12 以上 + uv
- 設定: `~/.config/ai-monitor/settings.yaml`（`settings.yaml.example` をコピーして作成）
- 環境差分は `settings.{環境}.yaml` に書き、環境名は環境変数 `AI_MONITOR_ENV`（または起動引数 `--env`）で指定

## 結合テスト（フロント）

なし

## 結合テスト（バック）

pytest で実行。

**前提:**

- GitHub API（githubkit）/ モニター HTTP は Mock 化（実 API は叩かない）
- 対象は `設計図/バックエンド結合/` の詳細ファイルと 1:1（各ファイル冒頭の対応テストファイル）

**全件実行:**

```bash
uv run pytest tests/integration/
```

**個別ツール実行:**

```bash
uv run pytest {結合テストファイルパス}
```

## 単体テスト（フロント）

なし

## 単体テスト（バック）

pytest で実行。DB / API 依存なし。

**前提:**

- ファイル外 / プロセス外の依存は Mock（一時 git リポジトリ・実 tmux を使うテストは fixture が作成 / 破棄）
- 対象は `設計図/モジュール構成/**` の単体テスト表と 1:1

**全件実行:**

```bash
uv run pytest tests/unit/
```

**個別ファイル実行:**

```bash
uv run pytest {単体テストファイルパス}
```

## 外部疎通テスト

**実 API（GitHub）に接続して疎通を確認** するテスト。
sandbox（`shuhei1101/ai-monitor-e2e`）への書き込みという副作用があるため **通常の CI では実行しない**。

**前提:**

- `~/.config/ai-monitor/settings.e2e.yaml` に sandbox 用の `github_token` と sandbox プロジェクト（`repo` / `local_path`）を設定（キーは `設計図/設定/モニター.md` 参照）
- `AI_MONITOR_ENV=e2e` を付与して実行する（PAT・対象プロジェクトが sandbox 向けに切り替わる）
- fixture が CWD を sandbox プロジェクトの `local_path` へ切り替える（MCP ツールの対象リポジトリは CWD の git remote から解決されるため）
- 検証用の Issue / PR / ラベル / ブランチは fixture が作成し、テスト後にクローズ / 削除する
- 誤実行防止のため `--run-external` フラグ必須（フラグなしだと skip される）
- テスト対象と確認観点は `設計図/モジュール構成/MCP/GitHub操作.md` の各ツールの `#### 疎通テスト` 表を参照

**全件実行:**

```bash
AI_MONITOR_ENV=e2e uv run pytest tests/external/ --run-external
```

**個別実行:**

```bash
AI_MONITOR_ENV=e2e uv run pytest {外部疎通テストファイルパス} --run-external
```
