# データフロー作成判定 チェックシート

`設計図/データフロー/README.md` のデータ一覧に載せる各データについて、DFD を作るかどうかを判定する。
以下の項目を上から確認し、**1 つでも YES があれば作成対象**、全部 NO なら作成不要。

## チェック項目

| 項目 | 補足 |
| --- | --- |
| 外部から入ってくるデータである（外部 API / ユーザー入力 / センサー / ファイルアップロード） | 発生源が システム外部 |
| 外部に出ていくデータである（外部 API / 通知 / 配信 / エクスポート / メール送信） | 出口が システム外部 |
| 複数のプロセス / モジュール / LLM / DB / 外部サービス を経由する | 単一関数内で完結しない |
| 変換 / 加工 が複雑である（フォーマット変換 / 集約 / 派生 / 分岐） | 単純代入 / コピー ではない |
| インシデント時に「どこで化けたか」を追いたい対象である | 障害調査対象 |
| セキュリティ / 個人情報 の観点で扱いを可視化しておきたい | PII / 認証情報 / 決済情報 等 |
| 別データの中間産物ではなく、独立した業務的意味を持つ | 中間産物なら親データ DFD に含めれば済む |
| 実装関数の局所変数ではなく、システムレベルで意味を持つ | 局所変数レベルなら不要 |
