# ユースケース要件 観点: 新規

新規作成系 UC（レコード新規登録 / 投稿 / 申請 等）で歩く観点。
`## ユースケース要件` の草案作成時に、以下カテゴリを 1 つずつ走査して該当行を要件として立てる。

## 入力

| No | 観点 | 洗い出す内容 |
| --- | --- | --- |
| 1 | フィールド一覧 | 入力可能な全項目 |
| 2 | デフォルト値 | 各項目の初期値（前回値 / 現在時刻 / ログインユーザー 等） |
| 3 | 必須 / 任意 | 各項目の必須指定 |
| 4 | 入力方式 | テキスト / セレクト / 日付ピッカー / リッチテキスト / ファイル |
| 5 | 依存フィールド | 別項目の値によって選択肢が変わる項目 |
| 6 | プリフィル | URL クエリ / 前レコードから複製 / テンプレートから |

## バリデーション

| No | 観点 | 洗い出す内容 |
| --- | --- | --- |
| 1 | フィールド単位 | 各項目の型 / 上限 / 下限 / 正規表現 |
| 2 | フォーム全体 | 項目間の相関チェック |
| 3 | 一意性 | 他レコードとの重複禁止項目 |
| 4 | 検証タイミング | 入力時 / フォーカスアウト時 / 送信時 |
| 5 | エラー表示位置 | インライン / トップサマリ / モーダル |

## 保存

| No | 観点 | 洗い出す内容 |
| --- | --- | --- |
| 1 | 二重送信防止 | 送信中のボタン無効化 / 冪等キー |
| 2 | 下書き保存 | 作成途中の一時保存を許すか |
| 3 | 送信後の初期化 | 「続けて作成」時のフォーム初期化方針 |
| 4 | 関連レコード生成 | 作成と同時に自動生成される付随レコード |

## フィードバック

| No | 観点 | 洗い出す内容 |
| --- | --- | --- |
| 1 | 成功時 | トースト / モーダル、表示時間 |
| 2 | 成功後遷移 | 詳細に遷移 / 一覧に戻る / 続けて新規作成 |
| 3 | 失敗時 | メッセージ表示、原因別粒度 |

## キャンセル

| No | 観点 | 洗い出す内容 |
| --- | --- | --- |
| 1 | 未保存離脱の確認 | ブラウザ戻る / タブ閉じ 時の確認ダイアログ |
| 2 | 変更検知 | 未入力の場合は確認をスキップ |

## アクセス制御

| No | 観点 | 洗い出す内容 |
| --- | --- | --- |
| 1 | 作成権限 | 誰が作成できるか（全員 / ロール別 / 招待制） |
| 2 | 数量制限 | 1 ユーザーあたりの作成上限 |
| 3 | レートリミット | 短時間の連続作成防止 |
