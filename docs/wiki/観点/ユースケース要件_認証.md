# ユースケース要件 観点: 認証

認証系 UC（ログイン / サインアップ / ログアウト / パスワードリセット / 二要素認証 等）で歩く観点。
`## ユースケース要件` の草案作成時に、以下カテゴリを 1 つずつ走査して該当行を要件として立てる。

## 認証方式

| 観点 | 洗い出す内容 |
| --- | --- |
| 方式 | ID + パスワード / メールマジックリンク / SSO / パスワードレス |
| 複数併用 | 複数方式の同時提供の可否 |
| 二要素認証 | 有無、方式（TOTP / SMS / メール / セキュリティキー） |
| パスワードポリシ | 長さ / 文字種 / 過去使用禁止 |

## 入力

| 観点 | 洗い出す内容 |
| --- | --- |
| フィールド | 認証情報の入力項目 |
| 形式チェック | メール形式 / パスワード最小要件 |
| 補助 UI | パスワード表示切替 / 大文字ロック警告 |

## セッション

| 観点 | 洗い出す内容 |
| --- | --- |
| 有効期限 | セッション寿命、アイドルタイムアウト |
| 記憶（remember） | 「ログイン状態を維持」オプション |
| 複数デバイス | 同時ログインの許可 / 制限、他デバイスのログアウト操作 |
| 手動ログアウト | ログアウト動線、全デバイスからログアウト |

## セキュリティ

| 観点 | 洗い出す内容 |
| --- | --- |
| レートリミット | 短時間の失敗回数制限 |
| ロックアウト | N 回失敗時のロック条件・解除方法 |
| CAPTCHA | 疑わしいアクセスの検知 |
| エラー粒度 | 「ID または PW が違います」等の情報漏洩防止文言 |
| パスワード保管 | ハッシュ方式、リセット時の再認証 |

## フィードバック

| 観点 | 洗い出す内容 |
| --- | --- |
| 成功時 | 遷移先（元ページ復帰 / ホーム / 初回オンボーディング） |
| 失敗時 | メッセージ、リトライ動線 |
| 復旧動線 | パスワード忘れ / アカウント復旧 |

## 監査

| 観点 | 洗い出す内容 |
| --- | --- |
| ログイン履歴 | 直近のログイン日時 / IP / デバイス の表示・保存 |
| 不審アクセス通知 | 新デバイスからのログインをメール等で通知 |
| 監査ログ | 認証イベント（成功 / 失敗 / ロック）を残す粒度 |
