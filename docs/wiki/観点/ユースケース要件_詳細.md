# ユースケース要件 観点: 詳細

詳細系 UC（1 レコードの詳細画面 / プレビュー / モーダル詳細 等）で歩く観点。
`## ユースケース要件` の草案作成時に、以下カテゴリを 1 つずつ走査して該当行を要件として立てる。

## 表示

| 観点 | 洗い出す内容 |
| --- | --- |
| 表示項目 | 見せる全フィールド、順序、グルーピング |
| 関連情報 | 関連レコード（コメント / 履歴 / 添付）を同画面に出すか、別画面か |
| 折りたたみ | 長文 / 大量関連情報の折りたたみ・展開 |
| フォーマット | 日付 / 金額 / 状態バッジ 等の表示形式 |
| 空フィールド | 値が無い項目の表示（「未設定」/ 非表示 / プレースホルダー） |

## 状態

| 観点 | 洗い出す内容 |
| --- | --- |
| 初期ロード | スケルトン / スピナー / キャッシュ表示 |
| 存在しない | 該当 ID が無い時の表示（404 相当） |
| 削除済み | 論理削除されたレコード表示の有無 |
| 取得エラー | 取得失敗時のメッセージ・リトライ動線 |
| 部分失敗 | 主データ取得成功 + 関連情報取得失敗 の見せ方 |

## 操作

| 観点 | 洗い出す内容 |
| --- | --- |
| 編集動線 | 編集画面遷移 / インライン編集 / モーダル |
| 削除動線 | 削除ボタン位置、確認導線 |
| 副次アクション | 複製 / 共有 / お気に入り / エクスポート 等 |
| 状態遷移 | ステータス変更（承認 / 却下 / 公開）の要否 |

## ナビゲーション

| 観点 | 洗い出す内容 |
| --- | --- |
| 戻る動線 | 一覧に戻るリンク、パンくず |
| 前後移動 | 前のレコード / 次のレコード への移動 |
| 直接 URL 共有 | 詳細 URL を共有された時の挙動（権限チェック等） |

## リアルタイム性

| 観点 | 洗い出す内容 |
| --- | --- |
| 他ユーザー更新 | 表示中に他人が更新した時の反映方法（自動 / 通知 / なし） |
| 楽観的ロック検知 | 編集画面へ移った時の版チェック |

## アクセス制御

| 観点 | 洗い出す内容 |
| --- | --- |
| 閲覧権限 | 誰が詳細を見られるか（本人のみ / チーム / 全員 / ロール別） |
| 表示項目の権限差 | ロールで見える項目 / マスクされる項目 |
| 操作権限 | 編集 / 削除 / 副次アクション を実行できるロール |
