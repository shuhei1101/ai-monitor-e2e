# ユースケース要件 観点: 編集

編集系 UC（既存レコードの更新 / プロフィール編集 / 設定変更 等）で歩く観点。
`## ユースケース要件` の草案作成時に、以下カテゴリを 1 つずつ走査して該当行を要件として立てる。

## 入力

| No | 観点 | 洗い出す内容 |
| --- | --- | --- |
| 1 | フィールド一覧 | 編集可能な全項目 |
| 2 | 既存値の取得 | 初期表示時に既存値をロード |
| 3 | 必須 / 任意 | 各項目の必須指定 |
| 4 | 入力方式 | テキスト / セレクト / 日付ピッカー / リッチテキスト / ファイル |
| 5 | 依存フィールド | 別項目の値によって選択肢が変わる項目 |
| 6 | 読み取り専用 | 表示するが編集不可の項目（ID / 作成日 等） |

## バリデーション

| No | 観点 | 洗い出す内容 |
| --- | --- | --- |
| 1 | フィールド単位 | 各項目の型 / 上限 / 下限 / 正規表現 |
| 2 | フォーム全体 | 項目間の相関チェック（開始 < 終了 等） |
| 3 | 一意性 | 他レコードとの重複禁止項目 |
| 4 | 検証タイミング | 入力時 / フォーカスアウト時 / 送信時 |
| 5 | エラー表示位置 | インライン（フィールド直下） / トップサマリ / モーダル |
| 6 | サーバー側検証 | クライアント検証を突破された場合の再検証と表示 |

## 保存

| No | 観点 | 洗い出す内容 |
| --- | --- | --- |
| 1 | 保存タイミング | 明示ボタン / 自動保存（debounce） / 離脱時に確認 |
| 2 | 楽観的ロック | 同時編集の検知と警告 |
| 3 | 二重送信防止 | 送信中のボタン無効化 / 冪等キー |
| 4 | 部分保存 | セクション別に保存できるか |
| 5 | 保存対象 | 差分のみ送信 / 全項目送信 |

## フィードバック

| No | 観点 | 洗い出す内容 |
| --- | --- | --- |
| 1 | 成功時 | トースト / モーダル / インライン、表示時間 |
| 2 | 成功後遷移 | 詳細に遷移 / 編集画面に留まる / 一覧に戻る |
| 3 | 失敗時 | メッセージ表示位置、原因別のメッセージ粒度 |
| 4 | ネットワーク切断 | オフライン時の挙動、リトライ |

## キャンセル

| No | 観点 | 洗い出す内容 |
| --- | --- | --- |
| 1 | 未保存離脱の確認 | ブラウザ戻る / タブ閉じ 時の確認ダイアログ |
| 2 | 元に戻す | 変更を破棄して元の値に戻す動線 |
| 3 | 変更検知 | 未変更の場合は確認をスキップ |

## 状態

| No | 観点 | 洗い出す内容 |
| --- | --- | --- |
| 1 | 初期ロード失敗 | 既存値取得失敗時の表示 |
| 2 | 編集不可状態 | 他ユーザーがロック中 / ステータスで編集不可 等 |

## アクセス制御

| No | 観点 | 洗い出す内容 |
| --- | --- | --- |
| 1 | 編集権限 | 誰が編集できるか（本人のみ / チーム / ロール別） |
| 2 | 項目別権限 | ロールで編集できる項目 / できない項目 |
| 3 | 監査ログ | 編集履歴を残すか、残す場合の粒度 |
