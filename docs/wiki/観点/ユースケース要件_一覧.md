# ユースケース要件 観点: 一覧

一覧系 UC（リスト / 検索結果 / タイムライン / ダッシュボードのカード列 等）で歩く観点。
`## ユースケース要件` の草案作成時に、以下カテゴリを 1 つずつ走査して該当行を要件として立てる。

## 表示

| 観点 | 洗い出す内容 |
| --- | --- |
| 表示項目 | リスト 1 件あたりに見せる項目（識別 / 副次情報 / メタ / 状態表示） |
| 表示密度 | コンパクト / 通常 / 詳細 のどれか、切替の要否 |
| 並び順デフォルト | 何をキーに、昇順 / 降順 |
| 件数表示 | 総件数 / 現在表示件数 / ページ番号 の表示要否 |
| 空データ表現 | 該当なし時のメッセージ・CTA（新規作成へ誘導 等） |

## フィルター

| 観点 | 洗い出す内容 |
| --- | --- |
| フィルター項目 | 何をキーに絞れるか（テキスト検索 / セレクト / 日付 / タグ） |
| 適用タイミング | 即時反映 / 明示ボタン送信 / debounce（何 ms） |
| 状態の永続化 | URL クエリ / ローカルストレージ / なし |
| 解除動線 | 個別解除 / 全解除、現在適用中の可視化 |
| 保存済みフィルター | よく使う条件を保存 / 呼び出し できるか |

## ソート

| 観点 | 洗い出す内容 |
| --- | --- |
| ソート可能項目 | どの列をキーにできるか |
| 複数キーソート | 対応するかどうか |
| 状態の永続化 | URL クエリ / なし |

## ページネーション

| 観点 | 洗い出す内容 |
| --- | --- |
| 方式 | 従来型ページャ / 無限スクロール / Load more ボタン |
| ページサイズ | デフォルト値、切替可否、切替時の状態リセット |
| 位置キープ | 詳細に飛んで戻った時にスクロール位置 / ページを保つか |

## 状態

| 観点 | 洗い出す内容 |
| --- | --- |
| 初期表示 | 何を表示するか（キャッシュ / スケルトン / スピナー） |
| ロード中 | 全画面 / 部分更新（フィルター変更時） の見せ方 |
| 空状態 | 「該当なし」時の表示 |
| エラー状態 | 取得失敗時のメッセージ・リトライ動線 |
| 部分エラー | 一部だけ失敗（関連情報取得失敗 等）した時の見せ方 |

## 操作

| 観点 | 洗い出す内容 |
| --- | --- |
| 行アクション | 詳細遷移 / 編集 / 削除 / お気に入り 等の要否 |
| 一括操作 | 複数選択 → 一括更新 / 削除 の要否、選択解除動線 |
| 主 CTA | 「新規作成」等の主ボタンの位置・ラベル |
| エクスポート | 表示結果を CSV / PDF 等で書き出す要否 |
| 自動更新 | ポーリング / WebSocket でリアルタイム更新するか |

## アクセス制御

| 観点 | 洗い出す内容 |
| --- | --- |
| 閲覧権限 | 誰が一覧を見られるか（本人のみ / チーム / 全員 / ロール別） |
| 表示項目の権限差 | ロールで見える列 / 見えない列 |
| 操作権限 | 一括操作 / 行アクションを実行できるロール |
