# 設定: モニター

`モニター` の設定値一覧。

- 置き場所は `~/.config/ai-monitor/settings.yaml`（`settings.yaml.example` をコピーして作成。MCP と共用）
- 環境差分は `settings.{環境}.yaml` に書き、環境名は環境変数 `AI_MONITOR_ENV`（または起動引数 `--env`）で指定する
- 実装は `shared/settings.py`（pydantic-settings）を参照
- ラベル定数の SoT は `plugins/ai-monitor/constants.env`

## ファイル一覧

| ファイル | 管理する値 | 補足 |
| --- | --- | --- |
| `~/.config/ai-monitor/settings.yaml` | 共通の設定値（秘匿情報含む） | git 管理外。サンプルは `settings.yaml.example` |
| `~/.config/ai-monitor/settings.e2e.yaml` | 外部疎通テスト・sandbox E2E 用の差分 | `AI_MONITOR_ENV=e2e` で読み込み |

## settings.yaml

| 論理名 | キー | 型 | 必須 | 既定値 | 説明 | 補足 |
| --- | --- | --- | --- | --- | --- | --- |
| GitHub Token | `github_token` | `str` | ✅ | - | GitHub API（githubkit）の認証 PAT | 秘匿。欠落時は起動エラー。fine-grained PAT に監視対象全リポジトリの Issues / PR / Contents の RW を付与 |
| 待受ポート | `port` | `int` | - | `8765` | `POST /completions` の待受ポート | bind 先は `127.0.0.1` 固定 |
| ポーリング間隔 | `poll_interval_sec` | `int` | - | `15` | GitHub ポーリングの周期（秒） | - |
| セッションタイムアウト | `session_timeout_min` | `int` | - | `30` | 処理中のまま超過したセッションを kill する閾値（分） | 待機中（assignee=ユーザー）は対象外 |
| heartbeat 間隔 | `heartbeat_interval_sec` | `int` | - | `60` | 生存確認の周期（秒） | 確認に失敗した（tmux にセッションが無い）場合は台帳から除去して tmux 側の実態に合わせる |
| 状態ファイル | `state_path` | `str` | - | `data/state.yaml` | セッション台帳の永続化先 | - |
| 監視対象プロジェクト | `projects[]` | `list` | ✅ | - | 監視するプロジェクトの一覧 | 子キーは下の行で展開 |
| 監視対象プロジェクト | `projects[].name` | `str` | ✅ | - | tmux セッション名・台帳キーに使う名前 | - |
| 監視対象プロジェクト | `projects[].repo` | `str` | ✅ | - | GitHub リポジトリ（`owner/name`） | - |
| 監視対象プロジェクト | `projects[].local_path` | `str` | ✅ | - | クローンのルート絶対パス | - |
| 監視対象プロジェクト | `projects[].wiki_base` | `str` | ✅ | - | 当該プロジェクトの Wiki raw URL のベース | スキルには SessionStart フックが `WIKI_BASE` として展開 |

## settings.e2e.yaml

| 論理名 | キー | 値 | 説明 | 補足 |
| --- | --- | --- | --- | --- |
| GitHub Token | `github_token` | sandbox 用の PAT | 疎通テストを sandbox の権限だけで実行する | 秘匿 |
| 監視対象プロジェクト | `projects[]` | sandbox（`shuhei1101/ai-monitor-e2e`）のみ | 疎通テストの fixture が `local_path` を CWD に使う | - |
