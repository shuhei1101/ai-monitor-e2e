# 監視対象追加

MCP ツール: `add_watch_targets`

エージェントが作成した派生 PR（PoC PR・Draft PR）の番号を、自セッションの監視面としてモニターの台帳に登録する。
以降その番号で起きたイベントは、新規セッションではなく登録元の既存セッションへ届く。
ツール内部でモニターの HTTP API（`POST /watch-targets`・ポートは設定の `port`）を呼ぶラッパー。

- 対応テストファイル: `tests/integration/mcp/test_add_watch_targets.py`

## インターフェース

### リクエスト

| パラメータ | 型 | 必須 | デフォルト | 説明 | 制限 | 補足 |
| --- | --- | --- | --- | --- | --- | --- |
| `agent_name` | str | ✅ | - | 報告するエージェント名 | - | セッションキーの片割れ |
| `number` | int | ✅ | - | 自セッションの主番号 | - | スキル起動時に渡された Issue / PR 番号 |
| `watch_numbers` | list[int] | ✅ | - | 監視面に追加する Issue / PR 番号 | 1 件以上 | 作成した派生 PR の番号 |

リクエスト例:

```json
{
  "agent_name": "architect",
  "number": 52,
  "watch_numbers": [60, 61]
}
```

### レスポンス

| フィールド | 型 | 説明 | 制限 | 補足 |
| --- | --- | --- | --- | --- |
| `ok` | bool | 受理したかどうか | - | 常に `true`（失敗は HTTP ステータスで表現） |

レスポンス例:

```json
{
  "ok": true
}
```

**補足:**

- 登録済み番号の再送は no-op で `ok: true` を返す（冪等）

### ステータスコード

| ステータスコード | 発生条件 | 補足 |
| --- | --- | --- |
| `200` | 正常受理 | 登録済み番号の再送（no-op）も `200` |
| `404` | 台帳に該当セッションがない | - |

## 制約

| 項目 | 制約 | 補足 |
| --- | --- | --- |
| タイムアウト | 制限なし | - |
| 受付元 | `127.0.0.1`（localhost）からのみ待受 | - |
| 対象プロジェクト | セッションは `project` + `agent_name` + `number` で特定する | `project` はツールが CWD のリポジトリから解決して自動付与 |

## フロー一覧

| 分類 | フロー名 | 概要 | 補足 |
| --- | --- | --- | --- |
| 正常 | 正常系 | MCP 委譲 → HTTP POST → セッション検索 → 監視面追加 | - |
| 異常 | 異常系（セッション不明・404） | 台帳に該当セッションがない | コールドスタート・台帳喪失 |

## 正常系

### セットアップ

| セットアップ | 説明 | 補足 |
| --- | --- | --- |
| Mock | モニターの `POST /watch-targets`（200 を返す） | - |
| セッション | 台帳に主番号 52 のセッションが登録済み | - |

### フロー

```mermaid
sequenceDiagram
  participant A as エージェント
  participant T as MCP ツール add_watch_targets
  participant MON as モニター

  A->>T: agent_name, number, watch_numbers
  T->>T: CWD のリポジトリから project を解決
  T->>MON: POST /watch-targets<br>（agent_name, number,<br>watch_numbers, project）
  MON->>MON: セッションの監視面番号一覧に追加 + 永続化
  MON-->>T: 200 OK
  T-->>A: ok: true
```

### 期待値

- 対象セッションの監視面番号一覧に `watch_numbers` が追加されている
- 戻り値が `ok: true`

## 異常系（セッション不明・404）

### セットアップ

| セットアップ | 説明 | 補足 |
| --- | --- | --- |
| Mock | モニターの `POST /watch-targets`（404 を返す） | - |
| セッション | 台帳に該当セッションが無い状態にする | 404 を決定的に誘発 |

### フロー

```mermaid
sequenceDiagram
  participant A as エージェント
  participant T as MCP ツール add_watch_targets
  participant MON as モニター

  A->>T: agent_name, number, watch_numbers
  T->>MON: POST /watch-targets
  MON-->>T: セッション不明（404）
  T-->>A: MCP ツールエラーとして返却
```

### 期待値

- MCP ツールエラー（404 の内容を含む）が返る
- エージェントは警告ログのみで続行できる（該当番号のイベントは新規セッションとして処理される）
