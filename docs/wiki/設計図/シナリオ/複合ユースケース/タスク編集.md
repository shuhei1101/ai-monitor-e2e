# タスク編集

既存タスクを一覧から選択して編集内容を保存する業務シナリオ。

- 対応テストファイル: `tests/e2e/複合ユースケース/task_edit.spec.ts`

## 正常シナリオ

### セットアップ

| セットアップ | 説明 | 補足 |
| --- | --- | --- |
| Mock | なし（実環境で実行） | - |
| `createUser` | ログイン中ユーザー A | - |
| `createTask` | userA が所有する編集対象タスク | - |

### フロー

```mermaid
flowchart TD
  U1([ユーザー A]) -->|タスク一覧画面を開き対象タスクを選択| UC1([タスク編集:正常シナリオ])
  UC1 -->|保存完了・一覧再取得| END([一覧に編集内容が反映された状態])

  click UC1 "../単一ユースケース/タスク編集.md#正常シナリオ"
```

### 期待値

- 一覧画面に編集後の内容が表示されている
- DB のタスクレコードが編集後の値になっている

## 異常シナリオ（他ユーザーが先に更新）

### セットアップ

| セットアップ | 説明 | 補足 |
| --- | --- | --- |
| Mock | なし（実環境で実行） | - |
| `createUser` | ログイン中ユーザー A | - |
| `createUser` | ログイン中ユーザー B | - |
| `createTask` | userA が所有し両者で編集する対象タスク | 編集対象タスク |

### フロー

```mermaid
flowchart TD
  subgraph FOCUS["検証対象: 先行更新後の後発編集で楽観的ロックが働くか"]
    subgraph PHASE1["ユーザー B"]
      UC1([タスク編集:正常シナリオ])
    end

    subgraph PHASE2["ユーザー A"]
      UC2([タスク編集:異常シナリオ<br>（他ユーザーが先に更新）]) -->|保存試行・楽観的ロック失敗 409| E1([競合エラー表示])
    end
  end

  PHASE1 -->|ユーザー切り替え・updatedAt 更新済み| PHASE2

  click UC1 "../単一ユースケース/タスク編集.md#正常シナリオ"
  click UC2 "../単一ユースケース/タスク編集.md#異常シナリオ他ユーザーが先に更新"
```

### 期待値

- ユーザー A に競合エラーが表示されている
- DB のタスクレコードが B の更新内容のまま（A の変更は保存されていない）
