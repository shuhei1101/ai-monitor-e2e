# リセット実行

resetter が不要化した Issue/PR の巻き戻し対象（子孫 Issue・PR・ブランチ・worktree・追記 Wiki）を洗い出し、ユーザー承認を経て全て削除・クローズする単一ユースケース。

対応エージェント: `resetter`

## 正常シナリオ

### セットアップ

| セットアップ | 説明 | 補足 |
| --- | --- | --- |
| Mock | なし（実環境で実行） | - |
| 対象 Issue/PR | ユーザーが手動で `確認:resetter` を付与 | 唯一のユーザー手動起動エージェント |
| 巻き戻し対象 | 子孫 Issue / Draft PR / worktree / PoC ブランチが存在 | 進行途中の状態 |
| マージ済み subsystem | 一部の subsystem PR は story ブランチへ merge 済み | 配下ブランチごと消えることの検証用 |
| assignee | 未設定 | エージェント起動条件 |

### フロー

```mermaid
sequenceDiagram
  actor U as ユーザー
  participant GH as GitHub
  participant ORC as モニター

  U->>GH: 対象 Issue に 確認:resetter 付与<br>（唯一のユーザー手動起動）
  ORC-->>GH: polling（確認ラベル + assignee なし を検知）
  create participant MON as resetter
  ORC->>MON: tmux セッション作成 + skill 起動
  participant REPO as リポジトリ
  activate MON
  MON->>GH: Sub-issue リンクを再帰走査・<br>巻き戻し対象一覧を対象 Issue にコメント +<br>議論中 付与 + assignee=ユーザー 設定
  deactivate MON

  loop 応答ループ（「これは残す」「別 Issue<br>に移譲」等の調整がある間）
    U->>GH: 対象 Issue に調整指示コメント +<br>assignee 外し
    ORC-->>GH: polling（ユーザー返信 + assignee なし を検知）
    ORC->>MON: 既存セッションへ送信
    activate MON
    MON->>GH: 巻き戻し対象一覧を更新 +<br>対象 Issue の assignee=ユーザー 再設定
    deactivate MON
  end

  U->>GH: 対象 Issue に削除方針回答 +<br>議論中 除去 + assignee 外し
  ORC-->>GH: polling（議論中 除去 + assignee なし を検知）
  ORC->>MON: 既存セッションへ送信（巻き戻し実行）
  activate MON
  MON->>GH: 対象 Issue の自分宛コメント一括 Resolve
  MON->>GH: 対応 PR close（gh pr close --delete-branch）
  MON->>REPO: worktree / ローカルブランチ / PoC ブランチ削除
  MON->>GH: 子孫 Issue を再帰的に close<br>（reason: not_planned）x 件数分
  MON->>GH: 対象 Issue 自身も close・<br>確認:resetter 除去
  deactivate MON
  ORC-->>GH: polling（close を検知）
  destroy MON
  ORC->>MON: 当該 Issue/PR の全セッションを解放（kill）
```

### 期待値

- 子孫 Issue（孫以下含む）が全て close（`reason: not_planned`）
- 配下の PR が close、リモート / ローカルブランチ・worktree が削除済み（story にマージ済みの subsystem の commit もブランチごと消える）
- 追記 Wiki はブランチ削除により消滅
- 対象 Issue/PR に `確認:*` ラベルが残っていない
- 対象 Issue の自分宛コメントが全て Resolve 済み
