# 軽微修正の直接マージ

quick-implementer が chore Issue の指示どおりに直接修正し、TDD・Wiki 更新・レビューをスキップして、push 内容のユーザー確認（応答ループ）を経てマージする単一ユースケース。

対応エージェント: `quick-implementer`

## 正常シナリオ

### セットアップ

| セットアップ | 説明 | 補足 |
| --- | --- | --- |
| Mock | なし（実環境で実行） | - |
| chore Issue | `layer:chore` + `確認:quick-implementer` 付きで存在 | - |
| 修正対象 | 軽微修正対象のファイルが存在（typo 等） | - |
| assignee | 未設定 | エージェント起動条件 |

### フロー

```mermaid
sequenceDiagram
  actor U as ユーザー
  participant GH as GitHub
  participant ORC as モニター

  Note over GH: chore Issue に<br>確認:quick-implementer 付与済み
  ORC-->>GH: polling（確認ラベル + assignee なし を検知）
  create participant MON as quick-implementer
  ORC->>MON: tmux セッション作成 + skill 起動
  participant REPO as リポジトリ
  activate MON
  MON->>REPO: worktree 作成 +<br>chore/{分類}/{変更内容} に修正 commit push
  MON->>GH: PR 作成（Draft にせず Ready 直行・<br>本文は 紐づく Issue + 概要）
  MON->>ORC: 作成した PR の番号を<br>自セッションの監視面として台帳に登録
  MON->>GH: chore Issue に確認依頼コメント<br>（PR リンク + 変更内容）+<br>議論中 付与 + assignee=ユーザー 設定
  deactivate MON

  loop 応答ループ（修正指示がある間）
    U->>GH: chore Issue に修正指示コメント +<br>assignee 外し
    ORC-->>GH: polling（ユーザー返信 + assignee なし を検知）
    ORC->>MON: 既存セッションへ送信
    activate MON
    MON->>REPO: 修正 commit push
    MON->>GH: chore Issue の<br>assignee=ユーザー 再設定
    deactivate MON
  end

  U->>GH: chore Issue の 議論中 除去 +<br>assignee 外し（承認）
  ORC-->>GH: polling（議論中 除去 + assignee なし を検知）
  ORC->>MON: 既存セッションへ送信（マージ実行）
  activate MON
  MON->>GH: chore Issue の自分宛コメント一括 Resolve
  MON->>GH: gh pr merge --squash --delete-branch
  MON->>REPO: worktree をローカルから削除
  MON->>GH: chore Issue close + ラベル除去
  deactivate MON
  ORC-->>GH: polling（close を検知）
  destroy MON
  ORC->>MON: 当該 Issue のセッションを解放（kill）
```

### 期待値

- master に修正 commit が入っている
- PR が merged 状態（Draft を経由していない）・本文に `## 紐づく Issue` と `## 概要` が記入されている
- chore Issue が close、worktree / ブランチが削除済み
- 自分宛コメントが全て Resolve 済み

## 異常シナリオ（マージ衝突）

### セットアップ

| セットアップ | 説明 | 補足 |
| --- | --- | --- |
| Mock | なし（実環境で実行） | - |
| chore Issue | `確認:quick-implementer` 付与済み・push とユーザー承認まで完了 | マージ実行フェーズ |
| master | chore ブランチと同一ファイルを変更する commit が先行して入っている | 衝突を誘発 |

### フロー

```mermaid
sequenceDiagram
  actor U as ユーザー
  participant GH as GitHub
  participant ORC as モニター
  participant MON as quick-implementer
  participant REPO as リポジトリ

  Note over MON: 既存セッションを継続利用<br>（承認後のマージ実行）
  activate MON
  MON->>GH: gh pr merge 失敗（マージ衝突）
  MON->>GH: chore Issue に相談コメント<br>（衝突内容 + 解消案）+<br>議論中 付与 + assignee=ユーザー 設定
  deactivate MON

  loop 応答ループ（解消方針の調整がある間）
    U->>GH: chore Issue に解消方針コメント +<br>assignee 外し
    ORC-->>GH: polling（ユーザー返信 + assignee なし を検知）
    ORC->>MON: 既存セッションへ送信
    activate MON
    MON->>REPO: 衝突解消 commit push
    MON->>GH: chore Issue の<br>assignee=ユーザー 再設定
    deactivate MON
  end

  U->>GH: chore Issue の 議論中 除去 +<br>assignee 外し（解消の確定）
  ORC-->>GH: polling（議論中 除去 + assignee なし を検知）
  ORC->>MON: 既存セッションへ送信（マージ再実行）
  activate MON
  MON->>GH: gh pr merge --squash --delete-branch
  MON->>REPO: worktree をローカルから削除
  MON->>GH: chore Issue close + ラベル除去
  deactivate MON
```

### 期待値

- master に修正 commit（衝突解消込み）が入っている
- PR が merged 状態
- chore Issue が close、worktree / ブランチが削除済み
